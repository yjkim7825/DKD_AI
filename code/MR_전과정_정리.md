# MR 전체 과정 (통합본) — FN1 실제 숫자로

**최종 목표**: "FN1 발현↑ 가 DKD를 **일으키나**(인과)?" — SNP을 지렛대 삼아 답한다.

전체 한 줄:
```
eQTLGen(Z) →〔변환·선별〕→ 노출 → DKD GWAS(원본 beta·SE)
   →〔rsID로 붙이기〕→ 기울기 →〔SE 가중〕IVW → OR → 두 데이터 재현 → 검증
```

---

## 📌 먼저 알아야 할 두 값

**Zscore** (eQTLGen이 주는 값)
- 연관성을 표준화한 점수 = **beta ÷ SE**
- 부호 = 방향(+발현 올림 / −발현 내림), 절댓값 클수록 연관이 강함
- 예) rs10932612 Z=17.69 → "아주 강하게 발현을 올림"
- ⚠️ Z만으론 발현이 "몇 단위" 바뀌는지(효과 크기 원단위)를 모름 → beta로 변환 필요

**beta** (효과 크기 = 회귀계수)
- "대립유전자 1개 늘 때 값이 얼마나 변하나"
- 노출 beta = 발현 변화량 / 결과 beta = DKD 로그오즈 변화량
- 부호 = 방향, 크기 = 효과 세기

**관계**: `Zscore = beta / SE` → Z와 (빈도 p·표본 N)만 있으면 beta·SE 역산 가능 (STEP 1-3 원리)

---

## STEP 1. 노출(도구변수) 만들기 — eQTLGen에서

**1-1. eQTLGen이 뭐냐 (노출 데이터 출처)**

31,684명 혈액에서 "각 SNP이 각 유전자 발현과 얼마나 연관되나"를 잰 대형 eQTL 데이터. 한 줄 = SNP 1개 × 유전자 1개. FN1 원본:

| SNP | 대립유전자 | Zscore | N |
| --- | --- | --- | --- |
| rs10932612 | C/T | 17.69 | 30,901 |
| rs17525860 | T/G | −9.23 | 31,569 |
| rs615857 | A/G | 6.10 | 31,684 |

→ ⚠️ eQTLGen은 **Zscore만** 줌 (beta·SE 없음) → 1-3에서 변환 필요

**1-2. SNP 선별 + 독립화 (p<5e-8 + LD 클럼핑)**

*어떤 값이 기준을 넘어야 하나 ↓*

**① p<5e-8 — eQTLGen의 `Pvalue` 열(SNP↔발현 연관 p값)이 이 값보다 작아야 함**

| SNP | Pvalue (SNP↔발현) | 5e-8보다? |
| --- | --- | --- |
| rs10932612 | 5.1e-70 | ✅ 훨씬 작음 |
| rs17525860 | 2.7e-20 | ✅ |
| rs615857 | 1.06e-9 | ✅ |

→ 이 Pvalue = "SNP이 발현과 연관된 게 우연일 확률". 2천만분의 1(5e-8)보다 작아야 통과 (약한 도구 방지)

**② LD 클럼핑 — SNP 쌍끼리의 `r²`(상관)이 0.001보다 작아야 함**

- **r²** = 두 SNP이 같이 움직이는 정도 (0~1). 1000G **EUR 참조패널**로 계산, `kb=10000` 창 안에서
- 남긴 SNP끼리 **r² < 0.001** 이어야 = 사실상 무관(독립)
- 한 덩어리에서 **Pvalue가 가장 센 SNP(lead)** 하나만 대표 채택 → 붙은 나머지는 버림 (한 증인 여러 번 증언 방지)

→ FN1 도구변수 **3개** 확정 (rs10932612·rs17525860·rs615857, 서로 r²<0.001)

**1-3. Zscore → beta/SE 변환 ("도구변수로 바꾸기")**

MR은 beta·SE가 있어야 계산됨. `Z = beta/SE` 관계를 역으로 풀어:
```
beta = Z / √(2·p·(1−p)·(N + Z²))     ( p = 대립유전자 빈도 )
SE   = 1 / √(2·p·(1−p)·(N + Z²))
```

| SNP | Zscore | → beta_발현 | → SE |
| --- | --- | --- | --- |
| rs10932612 | 17.69 | +0.143 | 0.008 |
| rs17525860 | −9.23 | −0.091 | 0.010 |
| rs615857 | 6.10 | +0.060 | 0.010 |

→ 이 **beta_발현 = 완성된 노출(도구변수)**: "SNP이 FN1 발현을 얼마나 바꾸나"

---

## STEP 2. 결과(outcome) 데이터 준비 — DKD GWAS

**2-1. 결과 데이터가 뭐냐**

DKD **환자 vs 대조군(정상)** 을 모아, SNP마다 **로지스틱 회귀**로 "이 대립유전자가 환자 쪽에 더 많나?"를 검정. (FinnGen=핀란드 환자/대조군 — 인종 데이터가 아니라 **질병 데이터**)

**2-2. beta·SE가 어떻게 나오나**

그 회귀가 SNP마다 두 개를 **동시에** 산출:
- **beta** = 대립유전자 1개 늘 때 DKD 로그오즈 변화 (효과)
- **SE(sebeta)** = beta의 오차막대 (표본 많고 대립유전자 흔할수록 작아짐)

→ 결과 데이터는 beta·SE가 **원본 열에 이미 있음** → 변환 없이 그냥 읽음 (eQTLGen만 Z라 변환했던 것과 대비)

FinnGen 원본 (같은 FN1 SNP):

| SNP | beta_DKD | SE |
| --- | --- | --- |
| rs10932612 | +0.009 | 0.021 |
| rs17525860 | −0.013 | 0.025 |
| rs615857 | −0.037 | 0.026 |

---

## STEP 3. 노출 + 결과 붙이기 (harmonise)

**같은 rsID로 매칭**해 한 줄에 beta_발현·beta_DKD를 나란히. 붙일 때 **대립유전자 방향 정렬**(양쪽이 같은 대립유전자 기준인지 맞추고, 아니면 부호 뒤집음) — "발현 올리는 대립유전자가 DKD도 올리나"를 바르게 보려고.

(아래는 outcome=GCST 기준 — 유의하게 나온 쪽)

| SNP | beta_발현 | beta_DKD |
| --- | --- | --- |
| rs10932612 | +0.143 | +0.248 |
| rs17525860 | −0.091 | −0.021 |
| rs615857 | +0.060 | +0.167 |

---

## STEP 4. SNP마다 기울기 = 인과 크기

```
기울기 = beta_DKD ÷ beta_발현   ("발현 1 올릴 때 DKD 얼마나?")
```
- rs10932612: 0.248÷0.143 = 1.73
- rs615857: 0.167÷0.060 = 2.80
- → 셋 다 양수 = "발현↑ → DKD↑"

---

## STEP 5. IVW로 합치기 (SE가 가중치로 쓰임)

기울기가 SNP마다 다르니 하나로 합침. 이때 STEP 2의 SE 등장:
```
가중치 = 1 / SE²   → SE 작은(정확한) SNP에 힘을 더 줌
```
정밀도 가중평균 → 합친 기울기 1.51 → **OR = exp(1.51) = 4.53**, p=0.012

```
✅ OR 4.53 > 1 = FN1 발현↑ → DKD 위험↑ (인과, 유의)
```

MR-Egger·Weighted median = 같은 데이터를 다르게 합쳐 검증 (4.34/4.58로 일치)

---

## STEP 6. 두 결과 데이터로 반복 (재현)

STEP 2~5를 FinnGen·GCST 각각에:
- FinnGen: beta_DKD가 다 작음 → OR≈1.04, **null**
- GCST: → OR 4.53, **유의**
- 두 데이터에서 방향 맞는 유전자만 → 방향충돌 4개 빼고 **366개**, DEG와 교집합 → **핵심 10개**
