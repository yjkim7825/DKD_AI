# 01_preprocessing — 전처리 (데이터 3종류 다 모음)

전처리는 "데이터 종류마다 한 번"이에요. 그래서 여기 3개로 나뉘고,
각 종류는 **한 번만** 전처리 → 이후 단계들이 그 결과를 **재활용**해요.

| 하위폴더 | 데이터 종류 | 하는 일 | 결과 → 쓰는 곳 |
| --- | --- | --- | --- |
| `bulk/`  | GEO 발현행렬 | probe변환·log2·정규화·라벨·ComBat | 발현행렬 → 02_deg, 03_pathway, 04_ml |
| `mr/`    | eQTL + GWAS | 노출 생성·클럼핑·harmonise | 노출 → 05_mr |
| `scrna/` | 단일세포 | QC·정규화·Harmony·세포주석 | 세포행렬 → 06_scrna |

## 핵심 원칙
- **같은 종류 안에선 전처리 1번** → 결과 재활용 (각 단계가 또 전처리 X)
- bulk 발현행렬 하나 만들면 → DEG·경로·ML 다 그거 씀
- 데이터 구조가 완전히 달라서(발현행렬/SNP/세포) 한 전처리로 셋을 못 합침

## 각 폴더 공통
- `R/` (기능 함수) + `run_*.R` (실행) + `output/` (결과)

## 실행
```
bulk:  source(".../01_preprocessing/bulk/run_preprocess.R")
mr:    (준비 예정)
scrna: (준비 예정)
```
