# DKD 재현 — 전체 파이프라인 정리 (전처리 → 단일세포)

> 형식: 각 단계마다 **목적 / 최종 입력 데이터 / 중간 과정 / 결과 / 결론**, 그리고 다음 단계로 넘어가는 **연결 서사**("이어서 ㅇㅇ을 확보하기 위해 ㅇㅇ을 진행한다").
> 이미지가 필요한 곳은 **[이미지]** 로 재현 출력 파일 경로를 적어둠. (경로 앞부분은 `code\R-reproduce\` 기준)

---

## 큰 그림 — 왜 이 순서인가

- bulk RNA-seq로 **다르게 발현되는 유전자(DEG)** 를 찾고 → 그게 **무슨 병리과정(경로)** 인지 해석하고 → 그 유전자가 **원인인지(MR)** 가리고 → **발현변화 + 인과 + 방향**이 다 맞는 후보만 교집합으로 남기고 → **머신러닝**으로 진단에 쓸 2개(FN1·ALDH2)로 압축하고 → **임상 신기능(eGFR) 상관**·**분자도킹(치료제)**·**단일세포(어느 세포·언제)** 로 검증한다.
- 한 층위만 보면 놓치는 것을 여러 층위가 **같은 답(FN1 위험 / ALDH2 보호)** 을 내는지 교차검증하는 게 이 연구의 핵심(multi-omics).

---

> 전처리는 분석마다 다르므로(bulk / eQTL / 단일세포), 각 단계 안에 **[전처리]** 블록으로 그 단계 전용 전처리와 최종 출력물을 보여준다.

## 1. DEG (차등발현 분석) — 탐색 범위를 수천 개로 좁히기

- **[전처리] bulk 발현행렬 만들기 (DEG 들어가기 전)**
    - **원자료 (뭘 넣나)**: bulk 5종 — GSE96804(학습, 사구체) · GSE104948·104954(검증) · GSE142025(정상/초기/말기 RNA-seq) · GSE30529(재검증). 마이크로어레이는 **probe ID**, RNA-seq는 **유전자 심볼**로 들어온다.
    - **전처리 과정**
        1. **probe → gene 매핑**: 장비별 매핑표로 probe를 유전자 이름으로 변환. 한 유전자에 probe 여러 개면 `avereps()`로 평균 → 행=유전자, 열=샘플.
        2. **log2 + 정규화**: 발현값이 수백~수만 배 차이라 log2로 안정화(값>100이면 raw로 보고 자동 변환). `normalizeBetweenArrays()`로 칩별 밝기(형광) 편차 통일.
        3. **그룹 라벨**: 각 샘플에 Control/Early/Late 부여(예: GSM123_Late).
        4. **병합 + ComBat 배치보정**: 공통 유전자만 `intersect`로 남겨 병합 → **ComBat(sva)** 으로 데이터셋·실험실 차이(배치)만 제거하고 질병 신호는 보존. (정규화=밝기 통일(질병 무관) vs ComBat=병원/데이터셋 편차만 제거(질병 차이 유지).)
    - **→ 최종 전처리 출력물**: '한 실험처럼' 비교 가능한 **통합 발현행렬(유전자×샘플)**. 이게 DEG·경로·ML의 공통 입력.
    - [이미지] 전처리 결과 통합행렬: 기존 노션 `스크린샷 2026-07-11 180859.png`(병합·ComBat 후).

- **목적**
    - 2만+ 유전자 중 정상·초기·말기에서 **다른 것만** 골라 이후 분석의 탐색 범위를 줄인다(1차 필터). DEG는 "상관"까지만 말한다(원인은 MR에서).
- **최종 입력 데이터**
    - 위 전처리의 **통합 발현행렬**(정상/초기/말기 샘플 × 유전자), 도구: **limma**.
- **중간 과정**
    - limma로 그룹 간 비교(moderated t) → 두 기준을 **동시** 만족만 DEG로 채택.
        1. **|logFC| > 0.585** = 1.5배 이상 변화. log2를 쓰는 이유는 증가·감소를 대칭으로 맞추기 위함(2배↑=+1, 2배↓=−1).
        2. **FDR < 0.05** = 뽑은 것 중 가짜 비율 5% 미만. Benjamini-Hochberg로 다중검정 보정.
    - 출력: DEG 목록 → 볼케이노·히트맵으로 표시.
- **결과 (논문 vs 재현)**

    | 비교 | 논문 | 재현 |
    | --- | --- | --- |
    | 말기 vs 초기 | 2,833 (↑1,557/↓1,276) | 3,314 (↑1,755/↓1,559) |
    | 말기 vs 정상 | 3,525 | 4,022 |
    | 초기 vs 정상 | 390 | 671 |
    | 패턴 | 진행할수록 급증 | 동일 |

    - 재현 수치가 조금 높은 건 threshold·전처리 세부 차이. **패턴(진행할수록 급증)은 동일**.
    - 방향: 말기vs정상 ↑2,008/↓2,014, 초기vs정상 ↑287/↓384.
    - [이미지] 볼케이노·히트맵: 기존 노션 `Fig2_A_volcano_LatevsEarly.png` / `Fig2_B_heatmap_top50.png` (재현 산출).
- **가장 많이 변한 유전자 (Top3, logFC)**

    | 비교 | Top3 |
    | --- | --- |
    | 말기 vs 정상 | FOSB(−5.5), NR4A1(−4.7), FOS(−4.2) |
    | 말기 vs 초기 | KLK1(−4.2), SRRM4(−4.0), CXCL6(+4.0) |
    | 초기 vs 정상 | FOSB(−6.8), FOS(−6.0), EGR1(−5.3) |

    - 관찰: 스트레스 반응 유전자(FOS·FOSB·NR4A·EGR1 등 **즉시초기유전자 IEG**)가 **초기부터 크게 감소** → DKD 초기 신호 특징.
- **변화 구간 3분류 (우리 확장 — 저자에 없음)**
    - DEG가 **언제 변하나**를 정상↔초기↔말기 3구간 조합으로 분류(저자는 2그룹만 봐서 이 분류가 없음).

    | 구간 | 정의 | 개수 | 대표 유전자 |

    | ② 둘 다(계속) | N→E ∩ E→L | 239 | FOS, EGR1, ATF3 (IEG 계속 감소) |
    | ③ 말기에만 | E→L만 변함 | 3,075 | (말기에 대규모 발현 붕괴) |

    - **대부분(3,075)이 말기에 새로 터짐**, 초기부터 계속 변하는 건 239개뿐.
    - [이미지] 3그룹 히트맵·3분류 볼케이노·벤: `02_deg\output\GSE142025_3group_combined\heatmap_3group.pdf` · `vol_3stage.pdf` · `venn_stage.pdf`.
- **검증셋 (GSE30529)**
    - DKD vs 정상 검증: **1,178개**(↑843/↓335). 논문엔 개수 명시 없음 → 재현값만. 패턴(진행할수록 급증) 재확인용.
- **저자 코드 vs 우리 코드 (재현하며 정리)**
    - **핵심 limma 로직은 100% 동일** (model.matrix→lmFit→makeContrasts→eBayes→topTable, 기준 |logFC|>0.585 & adj.P<0.05).
    - 저자는 **2그룹 1비교(DKD vs Control) 하드코딩** / 우리는 **함수 1개(`deg_two`)로 만들어** 3그룹(3비교)+검증셋에 재사용.
    - `avereps`(중복 유전자 평균) 시점만 다름: 저자는 DEG 단계에서, 우리는 **전처리(prep1)에서 이미** 수행(값 동일).
    - 우리가 **추가**한 것: TOP20 테이블 + 3그룹 통합 시각화(3그룹 히트맵·3분류 볼케이노·벤).
- **재현하며 확인된 점 (검증)**
    - 저번 재현과 대조: GSE142025는 3비교 모두 **100% 일치**(3,314 / 4,022 / 671) → **전처리 재현이 정확**하다는 증거. GSE30529만 전처리 방식(CEL→RMA) 차이로 미세 차이(1,282→1,178).
    - **통계 주의**: DEG 유의성은 논문 통계 파트의 "t-검정"에 대응하나, 엄밀히는 **limma `eBayes`(조정-t 통계량)** + **Benjamini-Hochberg FDR**(단순 t검정 아님).
- **결론**
    - 병 진행에 따라 발현이 뚜렷이 뒤집히는 수천 개 DEG 확보. IEG(FOS·NR4A 등)는 초기부터 감소, 나머지 대부분은 말기에 터짐. 단, 유전자 이름만으론 의미가 없다.

> **이어서**, 이 DEG가 **무슨 병리과정(경로)** 을 켰는지 해석하기 위해 **경로 분석**을 진행한다.

## 2. 경로 분석 — 유전자를 '기능' 단위로 해석

- **목적**
    - 유전자 낱개가 아니라 **경로(기능) 단위로 묶어** "어떤 병리과정이 활성화됐나"를 본다.
- **최종 입력 데이터**
    - DEG 목록 + 유전자셋: **Hallmark 50 · KEGG 186** (+ Table S2의 병리경로 6종: 산화스트레스·염증·EMT·노화·자멸사·자가포식).
- **중간 과정 (세 방법)**
    1. **ORA(과대표현)**: DEG가 특정 경로에 몰렸나 → 초기하분포 검정. (x축=Count, 색=p.adjust)
    2. **GSEA**: 전체 유전자를 순위 매겨 경로가 위/아래로 쏠렸나 → NES(정규화 강화점수). NES>0 활성, NES<0 억제, |NES| 클수록 강하게 쏠림.
    3. **ssGSEA(GSVA)**: 샘플/세포마다 경로 활성 점수화 → 단계별 변화 추적.
- **결과**
    - **ORA**: 초기엔 관련 DEG가 적어 경로가 약하고(Count 작음), 말기엔 **EMT·염증·면역(Allograft)** 에 DEG가 크게 몰림.
    - **GSEA**: 말기에서 면역·EMT·염증 곡선이 왼쪽에서 높이 솟음(ES ~0.8) = 강하게 활성.
    - **ssGSEA**: 염증·EMT·자멸사·노화 4개가 **초기 억제(파랑) → 말기 활성(빨강)** 으로 전환. 검증셋(GSE30529)에서도 재현.
    - [이미지] 기존 노션 `Fig2_C/D/E_hallmark_*.png`, `Fig2_F_GSEA_LatevsEarly.png`, `Fig2_H/I_ssGSEA_*.png` (재현 산출).
- **결론**
    - 말기일수록 EMT·염증·면역 경로 상위 활성. DKD는 고혈당이 유발하는 **여러 기전(염증·산화·EMT·노화·자멸사)이 얽힌** 병 → 여러 층위를 동시에 봐야 한다는 근거.

> **이어서**, 이 유전자들이 단지 결과가 아니라 **진짜 원인인지**를 가리기 위해 **멘델리안 무작위화(MR)** 를 진행한다.

## 3. MR (멘델리안 무작위화) — 상관을 넘어 '인과' 가리기

- **목적 (왜 하나)**
    - DEG는 "같이 움직인다(상관)"까지만 말한다. **치료 표적은 원인이어야** 하므로, 유전자 발현이 DKD를 **일으키는지**를 인과로 가린다.
    - 방법: SNP(유전변이)은 태어날 때 정해져 평생 안 변한다 → '자연이 만든 무작위 실험'의 **지렛대(도구변수)** 로 쓴다. "발현을 바꾸는 SNP"이 DKD도 바꾸면 → 발현이 원인.

- **한눈에 — 데이터가 두 종류라 헷갈림 정리**
    - **① 완제품 (메인)**: 저자가 beta·SE까지 다 계산해 준 표 → **FN1·ALDH2 딱 2개 유전자**뿐(VCF 2개 + Supplementary Table 8). 논문의 **핵심 두 마커 확정용**. 변환 없이 바로 씀.
    - **② 원본 eQTLGen (보조)**: **전체 유전자(수천 개)** 를 담은 대용량 원본. 단 beta가 없고 **Zscore만** 줌 → 빈도로 beta를 역산해야 씀. 용도는 (a) 전체에서 인과 유전자 **366개 스캔**, (b) 완제품 없이도 FN1 결론이 같은지 **검증**.
    - → 즉 "완제품 2개"는 FN1·ALDH2를 확정하는 **메인**, eQTLGen은 그 위·아래를 받쳐주는 **보조**. 결론(FN1 위험·ALDH2 보호)은 둘 다 같음.

- **전체 과정 (FN1 예시로 쉽게)**
    1. **FN1 발현을 바꾸는 SNP(도구변수) 찾기** — p<5e-8(우연 아님) + LD 클럼핑(붙어 다니는 건 대표 1개) → FN1은 **3개**(rs10932612·rs17525860·rs615857).
    2. **각 SNP의 기울기 구하기** — 기울기 = beta_DKD ÷ beta_발현 = "발현 1 올릴 때 DKD 로그오즈 얼마나". SNP마다 값 다름(1.18·0.16·1.87).
    3. **IVW로 하나로 합치기** — 정밀한(오차 작은) SNP에 힘을 더 주는 가중평균(w=1/SE²) → 합친 β.
    4. **방향 판정** — OR=exp(β). **OR>1 = 위험**(발현↑→DKD↑) / OR<1 = 보호. p<0.05면 유의.
    5. FN1 → OR **2.78**(>1) = **위험**으로 확정. (ALDH2는 같은 방식 14개 SNP → OR 0.673 = 보호)
    - > 이 과정을 셀마다 실제 데이터로 출력: `00_explore/05_mr_steps.ipynb` (메인=완제품, 부록=eQTLGen 역산).

- **[전처리] 도구변수 만들기 (bulk 전처리와 완전히 다름)**
    - **① 완제품 경로**: VCF에서 beta·SE·eaf 파싱 → Supp8 도구변수와 매칭(이미 p<5e-8+LD 완료) → 결과 GWAS와 harmonise(같은 rsID·대립유전자 방향 정렬). **끝. 역산 불필요.**
    - **② 원본 경로(보조)**: eQTLGen Z + 빈도(eaf) → `beta = Z/√(2p(1−p)(N+Z²))` 로 역산 → p<5e-8 + LD 클럼핑 → harmonise.
    - **→ 최종 출력물**: 유전자별 **도구변수(SNP) 표**(beta·SE·eaf 정렬 완료) = MR 본분석의 노출 입력.
    - [이미지] `04_mr\output\table.SNP.GCST.csv` · `exposure.FN1_ALDH2.csv`. 전처리 walkthrough: `00_explore/05_mr_preprocess.ipynb`.

- **최종 입력 데이터**
    - **노출(원인)**: eQTL — 발현을 조절하는 SNP (①완제품 VCF / ②eQTLGen 원본).
    - **결과(발병)**: DKD GWAS 2종 — **FinnGen R12**(핀란드 64,663명) · **GWAS Catalog GCST90435706**(영국 388,955명). 둘 다 유럽계로 통일(인종 교락 방지·두 집단 재확인).

- **결과 (논문 vs 재현)**

    | 항목 | 논문 Supp9 | 재현 ①완제품(메인) | 재현 ②eQTLGen(보조) | 판정 |
    | --- | --- | --- | --- | --- |
    | FN1 | OR 2.78, p=0.012 | **OR 2.78, p=0.012** (완전일치) | OR 4.5 (Z스케일 차) | 방향·유의 일치 |
    | ALDH2 | OR 0.673, p=0.032 | **OR 0.673, p=0.032** (완전일치) | — | 방향·유의 일치 |
    | FinnGen | null | null (FN1 p=0.79) | 동일 | 일치 |
    | 서사 | FinnGen null→GCST 유의 | 동일 | 동일 | 일치 |

    - **메인(①완제품)은 논문과 소수점까지 완전 일치.** ②eQTLGen은 절대값만 다르고(노출 단위·데이터셋 차이) 방향·유의는 동일 → 완제품 없이도 같은 결론 = 튼튼함 확인.
    - 전체 스캔(②): 5,430 유전자 × 두 결과 → FinnGen 214 ∪ GCST 161, 방향충돌 4개 제외 = **366개 인과 유전자**(교집합 단계로 넘김).
    - [이미지] 재현 forest plot: `04_mr\output\forest_MR_GCST.png` · `forest_MR_FinnGen.png` · `forest_MR_FigureD.png`.
- **결론**
    - FN1=위험(OR>1), ALDH2=보호(OR<1) 인과 방향 확인. 인과 유전자 366개 확보.

> **이어서**, "발현변화(DEG) + 인과(MR) + 방향일치"를 **다 만족하는 신뢰 후보**만 남기기 위해 **교집합**을 진행한다.

## 4. 교집합 → 후보 10개 — 세 조건을 다 만족하는 것만

- **목적**
    - 원인(MR) + 발현변화(DEG) + **방향 일치**를 모두 만족하는 유전자만 남겨 신뢰도를 끌어올린다.
- **최종 입력 데이터**
    - MR 인과 유전자 366개 + DEG 방향(고발현/저발현).
- **중간 과정**
    - **MR 위험(OR>1) ∩ DEG 고발현** → 위험 후보 (발현↑ + 위험 = 방향 맞음)
    - **MR 보호(OR<1) ∩ DEG 저발현** → 보호 후보 (발현↓ + 보호 = 방향 맞음)
- **결과 / 결론**

    | 구분 | 유전자 | 기준 |
    | --- | --- | --- |
    | 위험 (발현↑ → 위험↑) | CREB5, **FN1**, IFI44L, SYTL2, XAF1 | OR>1, p<0.05 |
    | 보호 (발현↑ → 위험↓) | **ALDH2**, CA1, CDKN1B, TSPYL5, VNN2 | OR<1, p<0.05 |

    - 위험 5 + 보호 5 = **후보 10개**.
    - [이미지] 벤 다이어그램(A)·방향일치(B/C): 기존 노션 교집합 그림.

> **이어서**, 이 10개도 진단에 쓰기엔 많으므로 **실제로 환자를 잘 구분하는 2개**로 압축하기 위해 **머신러닝**을 진행한다.

## 5. ML (머신러닝 진단모델) — 10개 → 2개(FN1·ALDH2)

- **목적**
    - 후보 10개를 진단에 쓸 소수로 추리고, "환자를 잘 구분하나" 성능으로 증명한다.
- **최종 입력 데이터**
    - 후보 10개 발현 + **학습셋 GSE96804 / 검증셋 GSE104948·104954**(과적합 방지 위해 분리).
- **중간 과정**
    1. **배치보정 + PCA (Fig 4A)**: 학습·검증셋을 ComBat으로 합친 뒤 PCA로 잘 섞였나 확인.
    2. **LASSO (Fig 4B)**: 불필요 유전자 계수를 0으로 눌러 핵심만 선택. λ는 10-fold CV로 최적화 → **10개 → 6개**.
    3. **ROC/AUC 필터 (Fig 4D–G)**: 검증셋 **AUC>0.8** 만 통과 → **6개 → FN1·ALDH2 2개**(나머지 4개 <0.7).
    4. **발현 방향 (Fig 4C)**: FN1↑ · ALDH2↓ → MR·DEG·ML 세 방법이 일치.
    5. **조합 성능 (Fig 4H–I)**: RF·SVM·GLM·XGBoost 중 **GLM 최고(검증 AUC 0.942)**, calibration·DCA 확인.
- **결과 (논문 vs 재현)**

    | 지표 | 논문 | R | Python |
    | --- | --- | --- | --- |
    | FN1 ROC-AUC (train/valid) | 0.911 / 0.911 | 0.909 / 0.915 | 0.909 / 0.915 |
    | ALDH2 ROC-AUC (train/valid) | 0.912 / 0.815 | 0.940 / 0.784 | 0.940 / 0.784 |
    | 최종 유전자 | FN1·ALDH2 | 동일 | 동일 |
    | 방향 | FN1↑ · ALDH2↓ | 동일 | 동일 |

    - 선별 방식: 논문 본문은 LASSO→4모델, 저자 코드는 LASSO∩SVM-RFE(세부 다름). **최종 유전자·방향은 일치**.
    - [이미지] `05_ml\output\ML_compare_AUC.png`, `A_ROC_FN1.pdf` · `A_ROC_ALDH2.pdf`, `A_PCA_batch.pdf`, `A_cvfit.pdf`(LASSO), `B_venn.pdf`.
- **결론**
    - 최종 바이오마커 = **FN1·ALDH2**. 진단 성능도 검증셋에서 우수(AUC>0.8).

> **이어서**, 이 두 유전자가 **실제 신장 기능(eGFR)** 과 연결되는지 확인하기 위해 **임상 상관(Nephroseq)** 을 진행한다.

## 6. 임상 상관 (Nephroseq) — eGFR와 연결되나 (사구체 피어슨)

- **목적**
    - FN1·ALDH2 발현이 실제 **신기능(eGFR, 높을수록 건강)** 과 연결되는지 임상 데이터로 검증.
- **최종 입력 데이터**
    - Nephroseq의 FN1·ALDH2 vs eGFR 상관 결과(파일 5-1~5-4).
- **중간 과정**
    - **피어슨 상관 r**(−1~+1) 계산. r<0이면 발현↑일수록 신기능↓(위험), r>0이면 발현↑일수록 신기능↑(보호).

    | 유전자 | 논문 | 재현 | 판정 |

    - **ALDH2는 신기능과 양의 상관(보호), FN1은 음의 상관(위험)** → MR 방향과 임상이 일치.

## 7. 분자 도킹 — 레스베라트롤(이중표적 치료 후보)

- **목적**
    - FN1을 낮추고(섬유화↓) ALDH2를 올리는(보호↑) **한 약(이중표적)** 을 찾아 진단을 치료로 잇는다(신약재창출).
- **최종 입력 데이터**
    - **DSigDB** 약물-표적 데이터 → FN1↓ 79개 + ALDH2↑ 35개 후보. 단백질 구조: FN1(PDB 3m7p) · ALDH2(PDB 8dr9).
- **중간 과정**
    - FN1↓ ∩ ALDH2↑ 교집합 5개 → **레스베라트롤** 선택 → 분자 도킹(약이 단백질에 잘 끼워지나 시뮬레이션). **Vina score**는 더 음수일수록 강한 결합(<−5 유의).
- **결과 / 결론**

    | 항목 | 논문 | 재현 | 판정 |
    | --- | --- | --- | --- |
    | 이중표적 약 | 레스베라트롤 | 후보목록(S13) 확인 | — |
    | Vina | −7.0(FN1) / −9.8(ALDH2) | 도킹은 웹툴(CB-Dock2)이라 미재현, 값 참조 | 참조 |

    - 레스베라트롤은 포도·적포도주 성분으로 안전성이 검증돼 **바로 임상 진입이 쉬운** 후보. 도킹 계산 자체는 웹툴 기반이라 우리는 **후보목록(S13)만 확인**.

> **이어서**, bulk는 조직 평균이라 "어느 세포·언제"를 모르므로, FN1·ALDH2가 **어느 세포에서 언제 변하는지**를 보기 위해 **단일세포 분석**을 진행한다.

## 8. scRNA (단일세포) — 어느 세포에서, 언제

- **[전처리] 진짜 세포만 골라 배치 정렬 (단일세포 전처리 — bulk·MR과 완전히 다름)**
    - **원자료 (뭘 넣나)**: 세포×유전자 **희소행렬**. 원시 데이터엔 **빈 방울(세포 없는 바코드)이 대부분**이라 그대로 못 쓴다. GSE131882는 유전자가 **Ensembl ID(ENSG)** 로 들어와 심볼 변환 필요, GSE209781은 10x 원시(679만 배럴코드).
    - **전처리 과정**
        1. **QC 필터**: 세포당 유전자 **300~5000**, **미토 비율 <10%** 로 죽은/빈 세포 제거. (`min.cells=3`으로 3세포 미만 유전자, `min.features=200`으로 유전자 200 미만 세포도 제거.)
        2. **정규화**: LogNormalize(scale factor 1e4) → 세포별 총량 보정.
        3. **HVG·PCA**: 변동 큰 유전자 2000개 → 30 PC로 요약.
        4. **Harmony 배치보정**: 샘플(6개)마다 생긴 배치 차이를 정렬. (bulk의 ComBat에 대응하는, 단일세포용 배치보정.)
        5. **클러스터(res=0.2) + 마커 세포주석(12종)** → PCT를 손상마커(VCAM1/HAVCR2/SPP1)로 PCT/dPCT 세분.
    - **→ 최종 전처리 출력물**: QC 통과 **세포×유전자 + 세포타입 주석**(초기 20,620 / 말기 18,817세포, 논문 20,220/18,816과 거의 일치). 이게 본분석 입력.
    - [이미지] 전처리 결과: `06_scrna\output\01.featureViolin.pdf`(QC 지표), `01.after_harmony.pdf`(배치보정), `01.Fig7A_tSNE_celltype.pdf`(주석된 세포아틀라스), `01b.Early_tSNE_celltype.pdf`(초기).

- **목적**
    - bulk 평균으로는 알 수 없는 **세포 종류·병기별** FN1·ALDH2 변화와 세포 간 통신을 본다.
- **최종 입력 데이터**
    - 위 전처리의 **QC 통과 세포×유전자 + 주석**. (원자료는 단일세포 3종: **GSE131882(초기, 신장)** · **GSE209781(말기, 신장)** · GSE266146(소변). 저자 코드는 말기 중심.)
- **중간 과정 (전처리된 세포로 하는 본분석)**
    - 세포비율, FN1·ALDH2 발현, **AddModuleScore·GSVA(경로 활성)**, **유사시간(손상 진행 축)**, **CellChat(세포 간 통신)**.
    - 초기(GSE131882)도 같은 파이프라인으로 추가 처리(01b), 플랫폼 교락을 피해 **데이터셋 내부 PCT vs dPCT**로 단계 비교(05).
- **결과 (실제 R 실행 — 정직 판정)**

    | 데이터 | 유전자 | PCT | dPCT | 방향 | wilcox p | 판정 |

## 9. 종합 결론

- **FN1 = 위험(섬유화 축)** / **ALDH2 = 보호(산화스트레스 축)** — 발현·경로·MR·ML·임상 상관이 **같은 방향**을 가리킴(교차검증 성공).
- 신장 손상은 정상 → 경증(dPCT1) → 중증(dPCT2)의 **연속 과정**이며, 중간 단계가 개입(치료)의 창.
- **레스베라트롤**이 FN1↓·ALDH2↑ 이중표적 후보로 제시됨(신약재창출).
- 재현 관점 요약: DEG·경로·MR·교집합·ML·임상 상관은 논문과 방향·서사 일치. 단일세포는 **ALDH2↓ 재현 성공, FN1은 snRNA 데이터 한계로 방향 확인 불가**(정직 보고). 도킹은 웹툴이라 값 참조.
