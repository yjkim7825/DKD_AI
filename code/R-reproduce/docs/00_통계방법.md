# 00. 통계 방법 — 논문 명시 vs 우리 재현 실제

> ⚠ **정직하게 구분**: 논문 통계 파트는 검정들을 *일반적으로 나열*만 함(어느 그림에 뭘 썼는지 콕 집지 않음). 우리 재현 **코드가 실제로 돌린 검정**은 아래 표대로다. "논문에 있으니 우리도 다 썼다"가 **아니다**.

## 1. 논문이 명시한 통계 (Methods "통계 분석", R v4.5.1)
- **피어슨 상관계수**: 바이오마커 발현 ↔ eGFR 연관성
- **그룹 간 비교**: 변수 특성에 따라 **t-검정 / 카이제곱 / 피셔 정확검정 / 분산분석(ANOVA)** 중 적절히
- **모든 검정 = 양측 검정(two-sided)**, 유의수준 **p<0.05**
- (양측검정 = 차이를 양방향 다 봄. 별도 분석이 아니라 위 검정들의 설정)

## 2. 우리 재현 코드가 실제로 쓴 검정 (grep 확인)

| 단계 | 실제 코드 함수 | 통계 | 논문 대응 |
| --- | --- | --- | --- |
| 02 DEG | `limma::eBayes` + `topTable` (BH-FDR) | **조정 t통계량**(moderated t) + FDR | 논문 "t-검정" 자리(엄밀히는 limma) |
| 03 경로 | `clusterProfiler` GSEA(순열)·ORA(초기하) + `p.adjust` | 농축 유의성 | (논문 통계 파트 밖) |
| 04 MR | `mr_ivw`·`mr_egger_regression`·`mr_weighted_median` / `mr_heterogeneity`(Cochran Q) / `mr_pleiotropy_test`(Egger 절편) | MR 인과·이질성·다면발현 | (MR 전용 검정) |
| 05 ML | `roc` + `ci.auc`(부트스트랩 CI) | ROC-AUC·신뢰구간 (가설검정 아님) | (진단 성능) |
| 06 scRNA | `wilcox.test` | **윌콕슨 순위합**(비모수) | 논문은 "t-검정"이라 했으나 우리는 **윌콕슨** (단일세포 표준) |
| 07 임상 | (계산 안 함) | 피어슨 **r값을 Nephroseq DB에서 그대로 사용** | 논문 피어슨 = Nephroseq 제공값 (논문도 사실 DB 조회) |
| 08 도킹 | Vina 점수 임계값(<−5) | 검정 아님 | — |

**우리 코드에 없는 것**: **카이제곱·피셔·ANOVA** — 재현 범위에서 사용 안 함.

## 3. 핵심 정리
- **연속·2그룹 → t검정 / 연속·3그룹+ → ANOVA / 범주(개수·비율) → 카이제곱(기대빈도 작으면 피셔) / 상관 → 피어슨** — 이게 논문이 "변수 특성에 따라"라고 한 선택 규칙.
- 우리 재현은 목적상 **limma(조정-t)·윌콕슨·MR검정·ROC**를 실제로 썼고, **피어슨은 Nephroseq DB 값**을 그대로 인용.
- 공통: **양측, p<0.05** (limma·MR·윌콕슨 등 우리가 돌린 검정도 양측·p<0.05 기준).

## 4. 논문 쓸 때 표기 팁
- 우리 재현 방법으로 기술하려면: "DEG는 limma 조정 t통계량+BH FDR, 단일세포 그룹비교는 Wilcoxon rank-sum(양측), MR은 IVW·MR-Egger·weighted median + Cochran Q·Egger intercept, eGFR 상관은 Nephroseq Pearson r 인용, 유의수준 p<0.05."
- 원 논문 그대로 인용하려면 논문 Methods 문장(t검정·카이제곱·피셔·ANOVA·피어슨, 양측, p<0.05)을 쓰되, **우리 코드가 실제 돌린 것과 다를 수 있음**을 알고 있어야 함.
