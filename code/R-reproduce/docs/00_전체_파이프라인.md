## Materials and methods

전체 흐름: **GEO(무엇이 변하나) → eQTL-MR(그게 원인인가) → 경로(어떤 기전) → ML(진단모델 FN1·ALDH2) → single-cell(어느 세포) → 임상(신기능) → 치료(약)**

---

### 1. GEO

- 단계별(정상 / 초기 / 말기)로 어떤 유전자가 차이 나는가
  - → 차이 나는 후보 유전자들(FN1 포함 수백 개)
- 사용 이유
  - 구조가 행=유전자 × 열=환자(정상/초기/말기)이고, 각 칸이 그 환자의 유전자 농도(발현량)
    - → 정상 vs 초기 vs 말기에서 어느 유전자 농도가 달라지는지 비교 가능
- 데이터 예시 (GSE142025, 유전자 × 환자)

  | geneNames | A11A_Late | A12A_Late | A13A_Late | A14A_Late |
  | --- | --- | --- | --- | --- |
  | A1BG | 6.87 | 6.95 | 6.82 | 7.39 |
  | A1CF | 10.92 | 11.27 | 12.10 | 11.53 |
  | A2M | 16.26 | 16.36 | 16.32 | 16.49 |

  ── 행(유전자) 17,184 × 열(환자) 36 / 각 칸 = 그 환자의 발현량(농도)
  → 열 이름의 `_Late`처럼 환자마다 단계 라벨이 붙어, 단계별 농도 비교가 됨

---

### 2. eQTL-MR

- 그 후보들 각각에 대해, eQTL SNP(유전 변이)가 그 유전자 발현을 높여서 DKD를 일으키는가 검증
  - → 통과한 게 FN1, ALDH2
- 사용 이유
  - eQTL은 행=SNP × 열=그 SNP이 발현을 얼마나 바꾸나(beta)
  - GWAS는 행=SNP × 열=그 SNP이 DKD를 얼마나 일으키나
    - → 같은 SNP에 대해 "발현 효과"와 "DKD 효과"가 따로 들어있음
    - → 그래서 두 숫자를 이어붙여 "발현→DKD" 인과를 계산할 수 있음
- 예 (FN1 도구변수 3개, 실제 값)

  | SNP | FN1을… | DKD를… | 이 SNP의 결론(비율) |
  | --- | --- | --- | --- |
  | **rs615857** | 높임 (+0.060) | 높임 (+0.037) | +0.63 (FN1↑→DKD↑) |
  | **rs17525860** | 낮춤 (−0.091) | 낮춤 (−0.013) | +0.14 (FN1↓→DKD↓, 같은 결론) |
  | **rs10932612** | 높임 (+0.143) | 낮춤 (−0.009) | −0.06 (반대, 모순) |

  - SNP마다 "발현 1 올릴 때 DKD 얼마나 변하나"라는 비율을 냄
    - → 각 SNP의 비율을 정밀도로 가중 평균(IVW)해서 최종 인과(OR·p)를 결정
  - 결과: FinnGen 기준 IVW **OR 1.04, p=0.79 → 비유의**(SNP 방향 안 맞음) = 논문과 일치(올바른 재현)
    - outcome을 GCST(영국 38만명)로 바꾸면 정렬되어 **유의(FN1 OR 2.78 = 논문 Supp9)**

### 3. 경로 (Hallmark·KEGG / Table S2)

- 관련 유전자들이 어떤 병리경로에 몰리나 → 세 방법으로 분석 (논문 Fig4)
  - **ORA**(C/D/E/G): DEG가 경로에 몇 개 걸리나 (Count 막대)
  - **GSEA**(F): 전체 순위에서 경로가 위/아래 몰렸나 (NES 곡선)
  - **ssGSEA**(H/I): 샘플마다 경로 활성점수 (히트맵)
  - → 셋 다 **EMT(섬유화)·염증·면역** 활성 = 진행 기전 (상세: `03_경로분석_논문vs재현.md`)
- 사용 이유
  - 구조가 각 줄 = 경로명 + 그 경로에 속한 유전자 명단
    - → "어느 유전자가 어느 경로 소속인지" 명단이 다 있음
    - → 그래서 우리 유전자들이 어느 경로에 몰리는지 대조할 수 있음
- 데이터 예시 (4-1 Hallmark, 경로별 유전자 명단)

  | 경로명 | 소속 유전자 수 | 유전자 (앞부분) |
  | --- | --- | --- |
  | HALLMARK_ADIPOGENESIS | 200 | ABCA1, ABCB8, ACAA2, … |
  | HALLMARK_ALLOGRAFT_REJECTION | 200 | AARS1, ABCE1, ABI1, … |
  | HALLMARK_ANDROGEN_RESPONSE | 101 | ABCC4, ABHD2, ACSL3, … |

  ── 한 줄 = 경로 1개 + 소속 유전자 나열 (경로마다 유전자 수가 달라 원래는 직사각형 표 아님)
  → 우리 유전자(FN1 등)가 어느 경로 줄에 들어있는지 찾아 기전 해석

### 4. 진단모델 (ML / 논문 Figure 4)

- DEG∩MR 후보 10개에서 **진단에 쓸 최소 유전자**를 골라 모델을 만든다 → FN1·ALDH2
- 사용 이유
  - 학습셋(GSE96804)에서 고르고 **다른 코호트(GSE104948/54)로 검증** → 우연·과적합 아닌 진짜 바이오마커 확인
  - 구조: 행=유전자 × 열=샘플(Control/DKD 라벨) → 발현으로 환자/정상을 얼마나 잘 나누나 학습 가능
- 흐름 (논문 본문)
  - 배치보정(sva ComBat) → **LASSO**(10개→6개) → **ROC-AUC>0.8 필터**(학습·검증 둘 다) → **FN1·ALDH2** → **4모델(RF·SVM·GLM·XGBoost)** 결합
- 예 (최종 결합모델 검증 AUC)

  | 모델 | 검증 AUC | 비고 |
  | --- | --- | --- |
  | **GLM** | **0.972** | 최고 (논문 0.942 재현·상회) |
  | SVM / RF / XGB | 0.88 / 0.85 / 0.83 | 모두 >0.8 |

  → FN1·ALDH2 두 개만으로 강한 진단력. 저자 업로드 코드(LASSO∩SVM-RFE)는 붕괴 → 논문 방식 채택 (상세 `05_ML_두방식비교.md`)

### 5. single-cell

- 어느 세포종류가 FN1·ALDH2를 발현하나
- 사용 이유
  - 구조가 행=세포 하나 × 열=유전자라, 각 칸이 세포 한 개의 발현
    - → 세포 하나하나의 농도가 다 들어있음
    - → bulk 평균으론 못 보는 "어느 세포가 발현하나"를 세포 단위로 구분 가능
- 데이터 예시 (GSE209781, 10x — 유전자 × 세포, 각 칸 = 발현 count)

  | 유전자 \ 세포(바코드) | AAACCCAAGAAACACT-1 | AAACCCAAGAAACCAT-1 | AAACCCAAGAAACCCA-1 |
  | --- | --- | --- | --- |
  | MIR1302-2HG (ENSG00000243485) | 0 | 0 | 0 |
  | OR4F5 (ENSG00000186092) | 0 | 0 | 0 |
  | FN1 (예시) | 0 | 2 | 0 |

  ── 크기: 유전자 36,601 × 세포 6,794,880 (QC 후 진짜 세포만 남김). 값은 대부분 0인 **희소행렬**(위 숫자는 형태 예시)
  → 세포마다 발현이 따로라 "섬유화 만드는 특정 세포"를 콕 집어냄

### 6. 임상 (Nephroseq)

- FN1·ALDH2 발현이 eGFR(신기능)과 상관있나
- 사용 이유
  - 구조가 행=분석결과 × 열=p값·Fold Change·r값
  - 발현과 eGFR의 상관계수(r)가 이미 계산돼 있음
    - → 그래서 발현이 높을수록 신기능이 나빠지는지를 바로 볼 수 있음
- 데이터 예시 ① 질병군 vs 정상 (5-1 FN1, **Fold Change**)

  | Dataset | Analysis Type | p-Value | Fold Change |
  | --- | --- | --- | --- |
  | Woroniecka Diabetes Glom | over expression | 0.000008 | 2.808 |
  | Ju CKD Glom | over expression | 0.000200 | 3.631 |
  | Woroniecka Diabetes TubInt | over expression | 0.000244 | 5.271 |

  → FN1이 DKD 질병군에서 2.8~5배 **과발현**(올라가 있음), p값도 유의

- 데이터 예시 ② 신기능(eGFR) 상관 (5-2 FN1, **r Value**)

  | Dataset | Analysis Type | p-Value | r Value |
  | --- | --- | --- | --- |
  | Ju CKD Glom | correlation | 7e-15 | −0.523 |
  | Woroniecka Diabetes TubInt | correlation | 5e-5 | −0.755 |
  | Woroniecka Diabetes Glom | correlation | 5e-4 | −0.680 |

  → r **음수** = FN1 높을수록 **신기능(GFR) 낮음** = FN1이 신장을 나쁘게 함
  → (ALDH2는 반대로 r **양수** = 높을수록 신기능 좋음 = 보호적)

### 7. 치료 (DSigDB / Table S13)

- FN1 낮추는 / ALDH2 높이는 약물 후보
- 사용 이유
  - 구조가 행=약물 × 열=약물명·표적유전자·효과방향(UP/DOWN)·출처
    - → 어떤 약이 어느 유전자를 올리고/내리는지 목록이 있음
    - → 그래서 FN1 내리는 약·ALDH2 올리는 약을 골라낼 수 있음
- 데이터 예시 (Table S13, 약물 × 표적유전자)

  | Drug | Gene | Type (효과 방향) | Source |
  | --- | --- | --- | --- |
  | ajmaline | FN1 | 0.0000122=DOWN(HL60) | CMAP |
  | alexidine | FN1 | 0.0000068=DOWN(HL60) | CMAP |
  | ethanol | ALDH2 | increases(activity) | CTD |
  | Dasatinib | ALDH2 | increases(expression) | CTD |

  ── 한 줄 = 약물 1개가 어느 유전자를 올리는지/내리는지. **FN1은 DOWN(낮추는) 약, ALDH2는 increases(높이는) 약**
  → MR 결론(FN1↑=나쁨, ALDH2↓=나쁨)에 맞춰 FN1 낮추는 약·ALDH2 높이는 약을 골라 신약재창출
