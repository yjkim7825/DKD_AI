# 05. ML — 두 방식 구현·비교 (논문 Fig4 vs 저자 코드)

같은 입력(후보 10개 + 논문 코호트)으로 **두 파이프라인을 실제 R로 돌려** 비교.
- **방식 A (논문 Figure 4):** LASSO(10겹 CV) → 학습·검증 **ROC-AUC > 0.8 필터** → 4모델(RF·SVM·GLM·XGBoost) 결합
- **방식 B (저자 업로드 코드):** LASSO **∩ SVM-RFE** 교집합 → 유전자별 ROC

입력: 학습 `data.train.paper.txt`(GSE96804, 61명) / 검증 `data.valid.paper.txt`(GSE104948+104954, 71명) — **논문 코호트와 동일**.
후보 10개(DEG∩MR): ALDH2 · CA2 · CDKN1B · CREB5 · FN1 · IFI44L · SYTL2 · TSPYL5 · VNN2 · XAF1.
코드: `05_ml/01_ml_feature.R`(방식 A) · `05_ml/ml_author.R`(방식 B) · `run_ml.R`(한번에). 결과: `05_ml/output/`.

---

## 0. 결론 먼저 — 논문 방식이 맞았다

| | 방식 A (논문) | 방식 B (저자 코드) |
| --- | --- | --- |
| 1차: LASSO | 6개 선택 | 6개 (동일) |
| 2차 선별 | **ROC-AUC>0.8** (학습+검증) | **SVM-RFE 교집합** |
| **최종 유전자** | ✅ **FN1 · ALDH2** (논문과 정확 일치) | ⚠ **VNN2 1개** (FN1·ALDH2 놓침) |
| 결합모델 | **GLM 검증 AUC 0.972** (논문 0.942 상회, GLM 최고 재현) | 없음 |
| 배치보정 | ✅ sva ComBat, batch=학습/검증 (논문 방법) | — |

**두 방식을 다 돌려본 결과, 저자 코드 방식(LASSO∩SVM-RFE)은 이 데이터에서 엉뚱한 VNN2 하나로 붕괴했고, 논문 방식(ROC>0.8 필터)은 FN1·ALDH2를 깔끔하게 복원했다. 그래서 논문 방식을 그대로 따랐다.**

## 1. 공통 1단계 — LASSO (동일)

10겹 교차검증(λmin) → 후보 10개 중 **6개** 생존:
`ALDH2, CDKN1B, FN1, IFI44L, VNN2, XAF1`

→ 논문 Fig 4B의 6개(`CDKN1B, ALDH2, FN1, XAF1, TSPYL5, VNN2`)와 **5개 일치** (우리는 TSPYL5 대신 IFI44L — 정규화·probe 매핑 경계 차이).

## 2. 방식 A — ROC-AUC>0.8 필터가 최종 결정 ✅

LASSO 6개를 학습·검증 **양쪽** ROC-AUC로 평가, 둘 다 0.8 넘어야 통과:

| 유전자 | 학습 AUC | 검증 AUC | 통과 |
| --- | --- | --- | --- |
| **ALDH2** | 0.940 | 0.807 | ✅ |
| **FN1** | 0.909 | 0.871 | ✅ |
| CDKN1B | 0.967 | 0.718 | ✗ (검증 부족) |
| VNN2 | 0.974 | 0.649 | ✗ (검증 부족) |
| IFI44L | 0.627 | 0.640 | ✗ |
| XAF1 | 0.778 | 0.630 | ✗ |

→ **최종: FN1 · ALDH2** — 논문과 **정확히 일치**.
핵심: "학습에선 최고(CDKN1B 0.967, VNN2 0.974)인데 검증에서 무너지는" 과적합 유전자를 **ROC 이중필터가 정확히 제거**. 이게 논문 방식의 힘.

### 4모델 결합 (FN1+ALDH2) — 학습 / 검증 AUC
**논문 방법(sva ComBat, batch=학습/검증, mod=NULL) 적용 후:**

| 모델 | 학습 | 검증 |
| --- | --- | --- |
| **GLM** | 0.980 | **0.972** ← 최고 |
| SVM(RBF) | 0.973 | 0.875 |
| RF | 1.000 | 0.847 |
| XGBoost | 0.997 | 0.825 |

→ **GLM이 검증에서 최고(0.972)** — 논문의 "GLM 최고(0.942)" 결론을 **그대로 재현**하고 수치도 논문을 상회. 두 유전자 결합만으로 강한 진단력.

**ComBat 설정 비교 (검증 AUC, `reference/combat_mod_compare.R`)**

| 배치보정 | 최종 유전자 | GLM | RF | SVM | XGB |
| --- | --- | --- | --- | --- | --- |
| **mod=NULL** (채택) | ALDH2+FN1 | **0.972** | 0.847 | 0.875 | 0.825 |
| mod=~그룹(라벨사용) | FN1만 | 0.897 | 0.821 | 0.889 | 0.755 |

→ `mod=~그룹`(검증 라벨을 보정에 사용)은 성능을 부풀리기는커녕 오히려 낮고 유전자도 1개로 줄었다. **mod=NULL이 더 낫고 라벨 미사용이라 깔끔** → mod=NULL 채택.

**※ "더 높은 AUC를 좇지 않는다"** — 앞서 코호트별 z-score로는 GLM 0.983이 나왔지만 논문 방법이 아니다. 방법을 검증 AUC 최대화 기준으로 고르는 것은 **정답지(검증셋)에 튜닝하는 것**이라 재현에서 금물. 0.972/0.974/0.983 차이는 잡음 수준이며 셋 다 논문(0.942)을 넘는다. 선택 기준은 "최고 숫자"가 아니라 **"논문 충실·방법 타당"** = ComBat + mod=NULL.

**※ 성능을 끌어올린 핵심 — 배치보정 (논문 방법: sva ComBat)**
학습(GSE96804)과 검증(GSE104948/54)은 **서로 다른 플랫폼**이라 값 스케일이 달라, 그냥 학습통계로 표준화하면 결합모델이 잘 전이되지 않았다(검증 GLM ≈ 0.82).
논문 Methods는 *"데이터셋 간 배치 효과는 sva 패키지로 보정"*(Fig 4A = 보정 후 PCA)이라고 명시 → 우리도 **동일하게 `sva::ComBat`** 로 3개 데이터셋을 배치로 놓고 보정한 뒤 LASSO·ROC·4모델을 돌린다(`01_ml_feature.R`의 A0 단계, `A_PCA_batch.pdf`로 PCA도 재현). 각 데이터셋이 Control·DKD를 모두 포함해 교락이 없어 안전.
ComBat 적용 결과 검증 GLM **0.972**(batch=학습/검증, mod=NULL). 전처리(prep3)는 검증 내부(104948↔104954)만 보정했고 학습↔검증은 미보정이라, 05에서 학습 vs 검증 배치를 ComBat으로 보정한다(이중보정 아님). ※ 초기 "코호트별 z-score"(0.98)는 논문과 방법이 달라 폐기.

## 3. 방식 B — LASSO ∩ SVM-RFE ⚠

- SVM-RFE(caret + RBF 커널) 최적 선택: **VNN2, CA2** (2개)
- **교집합(LASSO 6 ∩ SVM-RFE 2) = VNN2 하나뿐**

→ **FN1·ALDH2를 완전히 놓침.** SVM-RFE가 검증 일반화와 무관하게 학습 분리력만 보고 VNN2·CA2를 골랐고, LASSO와 겹치는 건 VNN2뿐이라 최종이 1개로 붕괴.
→ 저자 코드 방식은 **선별 안정성이 약하다**는 것이 실제 실행으로 드러남 (본문에서 논문이 4모델·ROC 방식으로 간 이유).

## 5. 한 줄 요약
> 논문 방식과 저자 코드 방식을 **둘 다 실제 R로 구현·실행**. 1단계 LASSO는 동일하지만, 2단계에서 갈렸다. 논문의 **ROC-AUC>0.8 이중필터**는 검증에서 무너지는 유전자(CDKN1B·VNN2)를 걸러 **FN1·ALDH2를 정확히 복원**했고, **논문 방법(sva ComBat, batch=학습/검증) 배치보정**을 적용하니 **GLM 결합 검증 AUC 0.972**로 논문(0.942)을 상회하며 "GLM 최고" 결론까지 재현. 반면 저자 코드의 **LASSO∩SVM-RFE**는 VNN2 하나로 붕괴해 FN1·ALDH2를 놓쳤다. **→ 논문 방식이 맞아 그대로 채택.**

※ 순수 재현이라 절대 AUC는 정식 환경과 소수점 차이 가능. 방향·결론(FN1 위험↑·ALDH2 보호↓, DEG·MR과 일치)은 변함없음.
