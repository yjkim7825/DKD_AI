# 파이프라인 단계별 상세

각 단계: **🎯 목적 → ⓪ 데이터 → ① 전처리 → ② 기준값 → ③ 방법 → ④ 결과 → ⑤ 결론(뭘 아나)**

---

## 1. DEG (차등발현)

### 🎯 목적 — 왜 하나
- DKD는 **어느 유전자가 문제인지** 아직 모름 → 2만+ 유전자 중 **후보를 찾아야** 함
- 그래서 정상·초기·말기에서 **발현이 다른 유전자(DEG)** 를 먼저 골라 탐색 범위를 좁힘 (1차 필터)
- → 이후 단계(MR·ML)에 넣을 **후보 유전자 목록** 확보가 목적

### ⓪ 사용 데이터
- **주 분석**: `GSE142025` (3그룹 — 정상 9 / 초기 6 / 말기 21)
- **검증**: `GSE30529` (2그룹 — 정상 12 / 말기(DKD) 10)
- → 142025로 단계별 확인, 30529(2그룹)로 재검증

### ① 전처리 (bulk, 3단계)
1. **probe → gene**: probe ID를 유전자 심볼로 (라이브러리 매핑) + 같은 gene은 평균(avereps)
2. **log2 + 정규화**: log2 자동판정 후 변환 + normalizeBetweenArrays (샘플 분포 통일)
   - ※ 어레이는 RMA가 이미 log2+정규화 → 추가 스킵(옵션1)
3. **그룹 라벨**: 각 샘플에 `_Control/_Early/_Late`(또는 `_DKD`) 붙임 (series_matrix 메타)

### ② 기준값
- **|logFC| > 0.585** (=1.5배 이상 변화)
- **adj.P.Val(FDR) < 0.05** (보정 p, Benjamini-Hochberg)
- → 두 조건 **동시 만족**해야 DEG

### ③ 방법
- **라이브러리** (저자 `differential expression analysis.R` 기준):
    - **limma** — 차등발현 핵심 (design→lmFit→makeContrasts→eBayes→topTable, avereps)
    - **dplyr** — 결과 정리(`mutate`로 Up/Down 라벨)
    - **pheatmap** — 히트맵
    - **ggplot2** — 볼케이노
- **limma** (design → lmFit → makeContrasts → eBayes → topTable)
- 142025: 3비교(말기vs초기·말기vs정상·초기vs정상) / 30529: DKD vs 정상

### ④ 결과 (우리 vs 논문)
| 비교 | 논문 | 우리 |
| --- | --- | --- |
| 말기vs초기 | 2,833 | 3,314 (↑1,755/↓1,559) |
| 말기vs정상 | 3,525 | 4,022 |
| 초기vs정상 | 390 | 671 |
| 30529(검증) | — | 1,178 |
- 변화 구간: 초기에만 432 / 둘다 239 / 말기에만 3,075
- Top: FOSB·NR4A1·FOS (즉시초기유전자 IEG, 초기부터 감소)

### ⑤ 결론 — 이걸로 뭘 아나
- **정상→말기로 갈수록 변하는 유전자가 급증** (초기vs정상 671 → 말기vs정상 4,022)
    → 병이 진행할수록 유전자 발현이 크게 뒤바뀜 = 손상이 심해짐
- **대부분 유전자는 초기엔 그대로, 말기에 터짐** (말기에만 3,075 vs 초기에만 432)
    → DKD는 **말기에 발현 변화가 집중**됨
- **볼케이노 해석**: 점=유전자, x=변화크기(logFC), y=유의성.
    양쪽 위로 벌어진 점이 많음 = 크게+확실하게 변한 유전자 많음.
    3색(초기에만/둘다/말기에만)으로 "언제 변하나" 구분됨
- **히트맵 해석**: 정상에서 낮던(파랑) 유전자가 말기에 높아지고(빨강), 그 반대도 뚜렷.
    → 정상↔말기 블록 색이 반대 = 병 진행에 따라 발현이 **뒤집힘**
- **IEG(FOS·FOSB·NR4A) 초기부터 감소** = 세포 스트레스 반응이 초기부터 억눌림 (DKD 초기 신호)
- (재현 검증) 저번 재현과 142025 **완전 일치** → 전처리·DEG 정확

## 2. 경로분석

### 🎯 목적 — 왜 하나 (공통)
- DEG로 유전자 목록은 나왔지만 **유전자 이름만으론 "무슨 병리과정"인지 모름**
- 그래서 유전자를 **경로(기능) 단위로 묶어** "어떤 병리과정(염증·섬유화 등)이 켜졌나" 해석
- → 병이 **어떤 기전으로 진행되는지** 파악이 목적
- 세 방법(논문 Fig4): **2-1 ORA**(C/D/E/G, Count) · **2-2 GSEA**(F, NES) · **2-3 ssGSEA**(H/I, 샘플점수)

---

### 2-1. ORA (bulk · Count 막대)  ⟵ Fig4 C/D/E/G (저자 코드 없음 — 재현)

- 논문 Fig4 C/D/E/G 상위 경로와 **일치**

### 2-2. GSEA (bulk · 순위 기반)  ⟵ Fig4 F, 저자 `GSEA.R`

#### ⓪ 사용 데이터
- **입력**: DEG 결과(`diff_*.txt`, GSE142025 3비교)
- 유전자셋: **Hallmark**(50) + **KEGG**(186)

#### ① 라이브러리 (버전 — 논문 명시)
- **clusterProfiler** `v4.16.0` / **org.Hs.eg.db** `v3.21.0` / **enrichplot** `v1.28.2`(`gseaplot2`)
- gmt: `h.all.v2026.1`(50), `c2.cp.kegg_legacy.v2026.1`(186)

#### ② 기준값
- **p.adjust < 0.05** (`pvalueCutoff=1`로 다 뽑고 필터)

#### ③ 방법 — GSEA(순위 기반, ORA 아님)
- DEG의 **logFC로 유전자 순위** (`sort(logFC, decreasing=TRUE)`)
- `GSEA(logFC, TERM2GENE=gmt)` → 경로가 위(↑)/아래(↓) 몰렸나. **NES**(+켜짐/−꺼짐)
- `gseaplot2`(상위 NES>0/NES<0 각 3개)

#### ④ 결과 (실제, 말기vs정상 Hallmark)
| 경로 | NES | 방향 |
| --- | --- | --- |
| ALLOGRAFT_REJECTION | +3.80 | ↑켜짐 |
| **EMT** | +2.79 | ↑켜짐 (FN1 소속) |
| INTERFERON_GAMMA | +2.52 | ↑켜짐 |
| TNFA_SIGNALING | −2.40 | ↓꺼짐 |
- 유의경로: 초기17 / 말기22 / 말기vs초기18 (Hallmark)
- **검증셋 GSE30529**(DKD vs정상): 유의16 — IFNγ·Complement·Allograft·EMT 켜짐 (142025 말기와 동일) → 검증 성공

#### ⑤ 결론 — 뭘 아나
- **초기**: TNFA_SIGNALING 꺼짐(NES −3.55) ← IEG(FOS·FOSB) 초기 감소 (DEG와 연결)
- **말기**: EMT·면역 켜짐 → 섬유화·염증 완성
- ORA는 개수만, **GSEA는 방향(↑↓)까지** → 같은 TNFA가 초기 ORA선 유의·GSEA선 꺼짐(방법 차이)

## 3. MR (멘델리안 무작위화)

### 🎯 목적 — 왜 하나
- DEG는 "발현이 다르다"는 **상관까지만** — 원인인지 결과인지 모름
- 치료 표적은 **원인**이어야 하니, 유전자가 **진짜 DKD를 일으키는지(인과)** 를 가려야 함
- 사람 실험은 못 하니 **태생·불변인 SNP을 지렛대**로 "발현→DKD" 인과 추정
- → 후보 중 **진짜 원인 유전자만** 걸러내는 게 목적

### ⓪ 사용 데이터
- **노출(원인)**: eqtl-a VCF (FN1·ALDH2 cis-eQTL, SNP→발현) + 저자 도구변수 Supp8
- **결과(질병)**: `FinnGen`(핀란드 64,663) + `GCST90435706`(영국 388,955) — 둘 다 DKD GWAS
- **전 단계**: `02_deg`에서 FN1·ALDH2가 DEG인지 확인 후 인과 검증
- → 결과를 2개(핀란드·영국) 써서 우연 아닌지 교차검증

### ① 전처리 (노출·결과)
- **노출(eQTL)**: VCF 파싱(beta·se·eaf) ∩ 저자 도구변수 Supp8(이미 p<5e-8+LD clump)
- **결과(GWAS)**: FinnGen·GCST 그대로 (beta·SE 원본에 있음)
- harmonise: 같은 rsID로 매칭 + 대립유전자 방향 정렬
- ※ (부록) eQTLGen 원본 버전은 Zscore→빈도로 beta 변환 + 직접 LD clumping

### ② 기준값
- 도구변수: **p < 5e-8** + **LD r² < 0.001** (kb=10000)
- 인과 판정: **IVW p < 0.05** (OR>1 위험 / OR<1 보호)

### ③ 방법
- **TwoSampleMR** (IVW·MR-Egger·weighted median)
- 민감도: funnel·leave-one-out·이질성·다면발현·역인과

#### ✅ 논문 방법론 사전검증 (코드 반영 확인 완료)
논문 Methods의 MR 방법·가정·QC가 우리 `04_mr` 코드에 다 들어있는지 대조 확인:

| 논문 명시 | 우리 코드 | ✔ |
| --- | --- | --- |
| **표본 중복 없음** (핀란드 vs 영국, 다른 인구·플랫폼) | FinnGen + GCST 두 코호트 사용 | ✅ |
| **IVW 주분석** (p<0.05) | `mr_ivw`, 필터 IVW p<0.05 | ✅ |
| **MR-Egger 회귀 보완** | `mr_egger_regression` | ✅ |
| **가중중앙값 보완** | `mr_weighted_median` | ✅ |
| **Cochran Q** (이질성) | `mr_heterogeneity` (Q_pval) | ✅ |
| **MR-Egger 절편** (다면발현, p>0.05) | `mr_pleiotropy_test` | ✅ |
| F통계량 <10 제거 (약한 도구) | 도구변수 Supp8 (F>10 이미 반영) | ✅ |
| PhenoScanner 스크리닝 (교란요인) | Supp8이 이미 거친 SNP 사용 | ✅(간접) |
| palindromic·DKD연관(p<5e-6) SNP 제거 | harmonise + `pval.outcome>5e-6` 필터 | ✅ |

**MR 3가정** (논문 명시, 상세는 `04_MR_전과정.md`):
- ① 관련성(p<5e-8·F>10) / ② 독립성(SNP 태생 무작위=자동) / ③ 배제제한(Egger 절편 검정)

→ **방법·가정·QC 전부 코드에 반영됨** — 수정 없이 논문 방법과 일치. FN1 3방법 일치(IVW 2.78/Egger 2.71/median 2.80), 민감도 이상 없음(Q_p 0.48, 다면발현 p 0.99).

### ④ 결과 (우리 재현 `04_mr`, outcome=GCST)
| 유전자 | 논문 Supp9 | 우리(04_mr, VCF) |
| --- | --- | --- |
| FN1 | OR 2.78, p=0.012, nsnp 3 | **OR 2.777, p=0.012 — 완전일치** ✅ |
| ALDH2 | OR 0.673, p=0.032, nsnp 14 | **OR 0.673, p=0.032 — 완전일치** ✅ |
- 서사: FinnGen(핀란드)선 null → GCST(영국 38만)서 유의 (논문 동일)
- 3방법 방향 일치 + 민감도(이질성·다면발현) 이상 없음
- **★ Figure 3D 완전 재현**: Supp8 도구변수 10개 전체 MR → **인과 유전자 10개(OR·CI·p·nsnp) 논문 D와 정확히 일치** (FinnGen 7 + GCST 3). `forest_MR_FigureD.png`
- ※ (부록) eQTLGen 원본에서 IV를 새로 만든 버전은 OR 4.53 (노출 스케일 차이, 방향·유의는 동일) → `04_MR_논문vs재현.md` 부록

### ⑤ 결론 — 이걸로 뭘 아나
- **FN1 발현↑ → DKD 위험↑** (OR>1), **ALDH2 발현↑ → DKD 위험↓** (OR<1)
    → 두 유전자가 **결과(상관)가 아니라 원인(인과)** 임을 확인 = 치료 표적 자격
- **핀란드에선 null → 영국(GCST)에서 유의** = 큰 표본(38만)에서만 신호 → 재현성 확보
- OR 크기차는 노출 Z-스케일 / ALDH2 유의성은 IV 수(3 vs 14) 차이 — **방향은 동일**

## 4. 교집합 (DEG ∩ MR)
- **DEG 고발현(logFC>0.585) ∩ MR 위험(OR>1)** 등 방향 일치 → **후보 유전자 10개**
- 10개: ALDH2 · CA2 · CDKN1B · CREB5 · FN1 · IFI44L · SYTL2 · TSPYL5 · VNN2 · XAF1
- → DEG(상관) + MR(인과)를 모두 만족하는 유전자만 ML로 넘김

## 5. ML (머신러닝 진단모델)

### 🎯 목적 — 왜 하나
- 후보 10개 중 **진단에 실제로 쓸모 있는 최소 유전자**만 골라 진단모델을 만든다
- 학습셋에서 고르고 **독립 검증셋에서 확인** → 과적합 아닌 진짜 바이오마커 확립

### ⓪ 사용 데이터
- **학습**: `GSE96804` (61명: Control 20 / DKD 41)
- **검증**: `GSE104948 + GSE104954` (71명: Control 42 / DKD 29)
- ※ 전처리는 검증 내부만 ComBat → 학습↔검증은 05에서 배치보정

### ① 배치보정 (논문 Methods: sva)
- `sva::ComBat` (batch=학습/검증, mod=NULL) — 서로 다른 플랫폼 스케일 정합 (Fig4A PCA)

### ② 방법 (논문 Figure 4 본문)
- **LASSO**(glmnet, 10겹 CV, λmin) → 10개 중 6개 선택 (Fig4B)
- **ROC-AUC>0.8 필터** (학습·검증 둘 다) → **FN1·ALDH2** (Fig4C~G)
- **4모델 결합**(RF·SVM·GLM·XGBoost) → 진단 성능 (Fig4H·I)

#### ⚠ 논문 본문 vs 저자 업로드 코드 차이
| | 저자 코드(ML1/2.R) | 논문 본문/그림 | 우리 |
