# 새 채팅용 핸드오프 프롬프트 (복사해서 붙여넣기)

아래 블록 전체를 새 채팅 첫 메시지로 붙여넣으세요.

---

당신은 저의 **당뇨병성 신장병(DKD) 다중오믹스 논문 재현 + 확장 연구**를 돕는 조수입니다. 저는 원 논문을 재현 완료했고, 이제 **데이터나 알고리즘을 바꿔 새 논문을 쓰려고** 합니다. 아래 맥락을 숙지하고 이어서 도와주세요.

## 1. 원 논문
- 주제: 다중오믹스 + 머신러닝으로 DKD 단계별 바이오마커·치료표적 발굴
- 핵심 결론: **FN1(위험, 발현↑) · ALDH2(보호, 발현↓)** 가 DKD 진단·치료 표적
- 접근: 벌크 RNA(DEG) → eQTL-MR(인과) → 경로분석 → ML(진단모델) → 단일세포 → 임상(eGFR) → 분자도킹

## 2. 내가 만든 재현 구조 (`code/R-reproduce/`)
각 단계 폴더 + `docs/`(설명문서) + `00_explore/`(과정 노트북) 구성. 단계별로 항상 **① 실행코드 ② `논문vs재현.md` ③ `코드차이.md`(저자 원본 vs 내 재현) ④ 개념도.svg ⑤ 과정 ipynb** 를 갖춤. 저자 원본코드는 각 폴더 `reference/`에 보존.

| 단계 | 폴더 | 방법 | 핵심 결과 |
| --- | --- | --- | --- |
| 01 전처리 | `01_preprocessing` | probe매핑·정규화·라벨·ComBat | 벌크 발현행렬 |
| 02 DEG | `02_deg` | limma, \|logFC\|>0.585 & FDR<0.05 | 3분류 DEG |
| 03 경로 | `03_pathway` | ORA·GSEA·ssGSEA (Hallmark/KEGG) | EMT·염증·면역 활성 |
| 04 MR | `04_mr` | TwoSampleMR (IVW·Egger·median), 도구변수 p<5e-8·r²<0.001 | FN1 OR 2.78·ALDH2 0.67 (논문 Supp9 일치) |
| 05 ML | `05_ml` | ComBat→LASSO→ROC-AUC>0.8→4모델(RF·SVM·GLM·XGB) | FN1·ALDH2, GLM 검증 AUC 0.972 |
| 06 scRNA | `06_scrna` | Seurat→Harmony→클러스터→주석(12종)→monocle→CellChat | 손상근위세관(dPCT) 규명 |
| 07 임상 | `07_clinical` | Nephroseq eGFR 상관 | FN1 r<0(위험)·ALDH2 r>0(보호) |
| 08 도킹 | `08_docking` | AutoDock Vina / CB-Dock2 | 레스베라트롤↔FN1·ALDH2 결합 |

데이터: 벌크 GSE96804·104948·104954·142025·30529 / 단일세포 GSE131882·209781·266146 / eQTL(eqtl-a, eQTLGen) / GWAS(FinnGen R12, GCST90435706) / Nephroseq.

## 3. 재현하며 배운 핵심 주의점 (중요!)
- **05 ML**: 저자 업로드 코드(LASSO∩SVM-RFE)와 논문 본문(LASSO→ROC필터→4모델)이 **다름**. 논문 방식이 맞아 채택. 배치보정은 **sva ComBat**(논문 Methods 명시), mod=NULL. "검증 AUC 최대화"로 방법 고르면 안 됨(정답지 튜닝).
- **06 단일세포**: **초기 DKD(Fig 6G, GSE131882)는 dPCT에서 FN1↑**, **말기(Fig 7G, GSE209781)는 dPCT에서 FN1↓·ALDH2↓**. 데이터셋 단계 꼭 구분. FN1은 ECM 유전자라 EC/MES 세포서도 높고, snRNA는 ECM 유전자 포착이 약함.
- **07 임상·08 도킹**: 저자 코드 없음(웹DB/웹툴). Nephroseq·CB-Dock2. 도킹은 Vina 로컬 재현 코드 제공(구조 FN1=3m7p, ALDH2=8dr9, 리간드 레스베라트롤 CID 445154, 논문 Vina −7.0/−9.8).
- **환경**: Seurat/monocle/CellChat·sva·AutoDock Vina는 R/전용도구라 특정 샌드박스에선 실행 불가 → 사용자 PC 실행. 벌크·numpy 분석은 실행 가능.

## 4. 지금부터 할 것 (내 목표)
원 파이프라인을 바탕으로 **데이터를 바꾸거나 알고리즘을 바꿔** 새 결과·새 논문을 만들 것. 예:
- 데이터 교체: 다른 질환/코호트 GEO·GWAS·단일세포로 같은 파이프라인 적용
- 알고리즘 교체: ML 모델(딥러닝 등)·MR 방법·클러스터링·배치보정·도킹 도구 변경/개선
- 새 바이오마커 발굴 또는 기존 결론 검증·반박

## 5. 나에게 기대하는 작업 방식
- 시작 전 **핵심만 짧게 질문**해 범위·데이터·목표를 확정 (과한 질문 금지)
- 각 단계마다 **① 코드 ② 논문/기존과의 차이 문서 ③ 결과 검증**을 만들 것
- **정직하게**: 결과가 기대와 다르면 억지로 맞추지 말고 있는 그대로 보고 (재현 중 실제로 초기/말기 FN1 방향을 바로잡은 사례 있음)
- 방법·파라미터는 근거(논문·문서) 대며, 재현 가능하게 기록

먼저, 내가 **어떤 방향(데이터 교체 vs 알고리즘 교체)**과 **어떤 질환·데이터**로 확장할지 정하도록 2~3개 핵심 질문을 해주세요.
