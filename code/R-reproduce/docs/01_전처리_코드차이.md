# 01. 전처리 코드 비교 — 저자 원본 vs 우리 재현

저자 `data preprocessing 1/2/3.R` ↔ 우리 `R-reproduce/01_preprocessing/bulk/R/prep1·2·3`

## 📂 파일 경로 (입력 → 결과)

**코드**: `R-reproduce/01_preprocessing/bulk/run_preprocess.R` (+ `R/prep1·2·3`)

**입력 (원본 데이터)**: `code/data/` 의 CEL / series_matrix (GEO 원본)

**결과 (단계별 출력)** — `R-reproduce/01_preprocessing/bulk/output/`
| 폴더 | 내용 | 파일 |
| --- | --- | --- |
| `01_genematrix/` | probe→gene 매핑 결과 | `GSE{96804,104948,104954,142025,30529}.genematrix.txt` |
| `02_normalized/` | log2·정규화·라벨 완료 (**최종 발현행렬**) | `GSE*.normalized.txt` |
| `03_merged/` | 검증셋 병합+ComBat | `valid_104948_104954.combat.txt` |

→ **다음 단계(DEG·경로) 입력 = `02_normalized/GSE*.normalized.txt`**

**한 줄 결론**: 세 단계 다 **핵심 알고리즘은 동일**. 차이는 **입력 방식·매핑 방법·칩 확장·안전장치**뿐이고, **최종 결과는 소수점까지 일치**(검증됨).

---

## 전처리 1 — probe → gene

### 핵심 로직 (동일)
| 단계 | 저자 | 우리 |
| --- | --- | --- |
| probe→gene 매핑 | `inner_join(ids, exprSet)` | `AnnotationDbi::select(db)` |
| 같은 gene 평균 | `avereps(ID=Gene Symbol)` | `avereps(ID=sym)` ✅ 동일 |
| 대표 유전자 | `str_split(" // ")[,2]` | db가 처리 |

### 차이
| 항목 | 저자 | 우리 |
| --- | --- | --- |
| **입력** | `series_matrix.txt.gz` (이미 처리된 probe값) | **CEL 원본**(원시 신호) |
| **정규화** | 이미 돼있음 | **우리가 RMA로** 직접 |
| **매핑표** | `GPL5175.txt`(파일 다운) | **라이브러리**(hgu133a2.db 등) |
| **칩 종류** | 표준 1종만 | affy/oligo/brainarray 3방식 확장 |

- 왜 CEL부터: 검증셋(104948/954)은 series_matrix가 없고 CEL만 있어서 통일
- 왜 라이브러리: GPL.txt가 데이터별로 없거나 달라서, db가 매핑표 내장(다운 불필요)

```
저자:  series_matrix → GPL.txt 매핑 → avereps → geneMatrix
우리:  CEL → RMA(정규화) → db 매핑 → avereps → geneMatrix
```

## 전처리 2 — log2 + 정규화 + 그룹 라벨

### 핵심 로직 (동일)
| 단계 | 저자 | 우리 |
| --- | --- | --- |
| log2 자동판정 | `qx[5]>100 \|\| ...` | 동일 |
| log2 변환 | `log2(rt+1)` | `log2(mat+1)` 동일 |
| 정규화 | `normalizeBetweenArrays` | 동일 |
| 라벨 붙이기 | `paste0(colnames,"_",Type)` | `paste0(colnames,"_",grp)` 동일 |

### 차이
| 항목 | 저자 | 우리 |
| --- | --- | --- |
| **라벨 소스** | **s1/s2/s3.txt**(수동 목록) | **series_matrix 메타** / 열 접두사 |
| 적용 범위 | GSE142025 3그룹 전용 | 어레이·RNA-seq 범용 |
| log2 전 처리 | 음수→0 (`rt[rt<0]=0`) | 없음 (log2 대상 아니라 무관) |

- s1/s2/s3 = 저자가 GEO 보고 손으로 만든 정상/초기/말기 샘플 목록 → **공유 안 됨**
- 우리는 series_matrix 메타(diabetic→DKD) / 열 접두사(A=Late/B=Early/N=Control)로 대체
- ⚠️ **옵션1**: 어레이는 RMA가 이미 log2+정규화 → prep2 정규화 **스킵**(저자는 이중정규화)

## 전처리 3 — 병합 + ComBat

### 핵심 로직 (완전 동일)
| 단계 | 저자 | 우리 |
| --- | --- | --- |
| 공통 유전자 | `Reduce(intersect, geneList)` | 동일 |
| 병합 | `cbind(allTab, rt[interGenes,])` | `do.call(cbind, ...)` 동일 |
| 배치 지정 | `rep(i, ncol)` | `rep(seq_along, sapply(ncol))` 동일 |
| **ComBat** | `ComBat(..., par.prior=TRUE)` | 동일 |

### 차이
| 항목 | 저자 | 우리 |
| --- | --- | --- |
| **입력** | 폴더의 txt **자동 스캔**(`dir()`) | 합칠 행렬 **리스트 전달**(명시적) |
| 안전장치 | 없음 | 1개면 ComBat 스킵 + 분산0 유전자 제거 |
| avereps | 병합 전 또 함 | prep1에서 이미 함 |

- 저자: 폴더에 넣는 파일로 "뭘 합칠지" 결정 / 우리: 코드에 명시(검증셋 104948+104954)
- 우리 안전장치로 에러 방지(1개 데이터·분산0 유전자)

---

## 3단계 총정리

| 단계 | 핵심 로직 | 우리가 다르게/추가한 것 |
| --- | --- | --- |
| **1 probe→gene** | avereps 동일 | CEL부터(RMA), 라이브러리 매핑, 칩 3방식 |
| **2 정규화·라벨** | log2·normalize 동일 | 라벨=메타/접두사, 범용화, 옵션1(이중정규화 제거) |
| **3 병합·ComBat** | ComBat 동일 | 입력=리스트(명시), 안전장치 |

> **결론**: 핵심 알고리즘은 저자와 동일. 우리는 **입력 유연화 + 방어 코드 + 칩 확장 + 라벨 오프라인화**만 추가. **최종 결과(FN1·ALDH2 값·그룹)는 5개 데이터 다 기존과 일치**.

# 심화 — 재현 차이 & "왜 문제 없나" (발표 대비)

각 항목: **저자 방식 → 우리 방식 → 왜 → 차이**

## A. probe → gene 매핑 방식
- **저자**: **주석 테이블(GPL/Brainarray CDF) 다운받아** 매핑
    - 104948·104954 = Brainarray 커스텀 CDF(폴더에 저자가 남김)
    - 원본 템플릿(data preprocessing 1.R) = series_matrix + GPL.txt
- **우리**: **Bioconductor 라이브러리**로 매핑 (다운 불필요)
    - 30529 → `hgu133a2.db` / 96804 → `hta20transcriptcluster.db`(oligo)
    - 104948·104954 → 저자 Brainarray CDF 그대로 따름
- **왜**: GPL.txt가 데이터별로 없거나 칩이 달라서, 표준 라이브러리(db)가 매핑표를 내장 → 다운 없이 처리. 저자 자료 있는 것만 그걸 따름
- **차이**: 방향 동일. 매핑 db에 따라 **유전자 수가 달라질 수 있음**(아래 B)

## B. GSE96804 유전자 수 (같은 데이터, 우리 내부 두 버전)
- **둘 다 GSE96804** (샘플 61명 동일) — 매핑 방식만 다름
    - 기존 재현: 33,720 유전자 / 새 재현(oligo): 23,910 유전자
- **왜 줄었나**: db가 **유전자 이름(심볼)이 붙는 것만** 남김 → 빠진 12,793개는 대부분 **비유전자**
    - 우리가 임의로 지운 게 아니라, **db가 이름을 못 붙여서 자동으로 빠진 것**

**"지워도 되나?" → 됨. 이유 3가지**

1. **표준 절차** — 마이크로어레이 분석에서 "이름 안 붙는 probe 제거"는 정식 방법 (저자도 함)
2. **어차피 못 쓰는 것** — DEG·경로·ML은 전부 **유전자 이름으로** 계산. 이름이 없으면(예: `AC000036.4`) 다음 분석에서 무슨 유전자인지 몰라 **아예 쓸 수가 없음** → 남겨도 무의미
3. **진짜 유전자가 아님** — 빠진 것들 종류:
    - **안티센스RNA(-AS1)**: 단백질을 안 만드는 RNA
    - **미지영역(AC00..)**: 아직 정체 모르는 유전체 좌표 (번호만)
    - **가성유전자(-P1)**: 기능 잃은 유전자 잔재
    → 셋 다 단백질을 만드는 진짜 유전자가 아님. 연구가 찾는 건 단백질 유전자(FN1·ALDH2)라 무관.

- **ML 성능 차이?** → 안 남. ML은 FN1·ALDH2 관련 후보 몇 개만 학습에 씀 → 두 버전 다 살아있음
- **DEG 정상/말기 편향?** → 안 남. 매핑은 probe 기준이라 그룹 무관 (게다가 DEG는 GSE142025=RNA-seq라 처음부터 심볼 → 매핑 이슈 자체 없음)
- **결론**: 유전자 수는 줄었지만 **진짜 유전자 손실 없고 결과·성능 영향 없음**. 새 방식이 더 깔끔.

## C. 정규화 / ComBat (이중 정규화)
- **저자**: 어레이도 RMA(이미 정규화) **후 normalizeBetweenArrays로 또 정규화** = 이중 정규화
- **우리**: 어레이는 RMA로 이미 정규화됨 → **추가 정규화 스킵(옵션1)**
- **왜**: 이미 RMA로 정규화된 걸 또 하는 건 중복 (저자는 데이터 합치기 대비/표준관행으로 넣은 듯)
- **차이**: 값 거의 동일, 우리가 더 깔끔. ComBat은 둘 다 sva 표준으로 동일

## D. 검증셋 샘플 필터 (104948·104954)
- **저자**: 196개 중 DN(당뇨)·LD(공여자)만 → 33/38개
- **우리**: 동일 (DN→DKD, LD→Control 필터)
- **왜**: 여러 질환 섞인 데이터라 DKD·정상만 선별
- **차이**: 없음 (저자 기준 따름)

## E. 그룹 라벨 붙이는 방법
- **저자**: **s1/s2/s3.txt** — 저자가 GEO 페이지 보고 **손으로 만든 샘플이름 목록** 3개
    (s1=Control, s2=Early, s3=Late) → 코드가 읽어 `GSM_그룹`으로 라벨
- **우리 (최종)**: 로컬 `series_matrix.txt.gz` **메타데이터에서 자동 추출** (diabetic→DKD)
    - (초기엔 `getGEO`(인터넷) 썼으나 오프라인 실패 → series_matrix 메타로 교체, 이제 인터넷 불필요)
- **왜**: s1/s2/s3 파일은 저자가 공유 안 함 + getGEO는 인터넷 필요 → 로컬 메타로 대체
- **차이**: **방법만 다르고 그룹 판정 결과는 동일**
    - ✅ 검증: 30529 DKD10/Control12, 96804 DKD41/Control20 — 기존과 그룹 수 일치
