# 08. 분자 도킹 (레스베라트롤 ↔ FN1·ALDH2) — 논문 vs 우리 재현

## 📂 파일 경로
- **코드**: `R-reproduce/08_docking/dock_vina.py` (AutoDock Vina 로컬 재현)
- **가이드**: `08_docking/README.md` (Vina + CB-Dock2 두 방법)
- **결과**: `08_docking/output/dock_summary.csv`, `*_out.pdbqt`

---

## 0. 논문이 한 것

DKD 치료제 후보로 **레스베라트롤**을 지목하고, 두 바이오마커 단백질에 **정말 결합하는지** 분자 도킹으로 검증.

| 항목 | 내용 |
| --- | --- |
| 리간드(약물) | **레스베라트롤** (PubChem CID 445154) |
| 표적 1 | **FN1** (PDB ID **3m7p**) |
| 표적 2 | **ALDH2** (PDB ID **8dr9**) |
| 도구 | **CB-Dock2** (blind docking, 자동 포켓탐지 + Vina 점수) |
| 전처리 | 수소 원자 추가, 물 분자 제거 |
| 결과 | **Vina 점수 FN1 −7.0 / ALDH2 −9.8 kcal/mol** (그림 5C·D) |
| 기준 | < −5.0 = 유의미한 결합 (음수 클수록 강함) |

→ 레스베라트롤이 FN1·ALDH2 **둘 다에 강하게 결합** = 이중 표적 치료 가능성.

---

## 1. 재현 — 두 가지 방법

도킹은 저자 **코드가 없다**(CB-Dock2는 웹툴). 그래서 두 경로로 재현:

### ① AutoDock Vina (로컬 코드) — `dock_vina.py`
- CB-Dock2의 핵심 엔진이 Vina라, **오픈소스 Vina로 같은 blind docking**을 로컬에서 코드로 수행.
- 논문이 준 **정확한 구조(3m7p·8dr9)** 와 **정확한 리간드(CID 445154)** 사용.
- 자동: 구조·리간드 다운로드 → 수소추가·물제거 → 박스 계산 → 도킹 → 점수.
- ⚠ CB-Dock2와 포켓탐지·스코어링 세부가 달라 **절대 점수는 다를 수 있음**. 재현의 판정은 "**< −5.0 강한 결합**이 두 표적 모두 나오는가"(경향 일치).

### ② CB-Dock2 (웹툴, 논문과 동일)
- 논문이 쓴 그 웹서버에 구조·리간드 올려 실행 → 논문과 **정확히 같은 방법**.
- 코드 기록은 안 되지만 논문값(−7.0/−9.8)과 직접 대조 가능.

---

## 2. 왜 "코드 재현"이 까다로운가
- 도킹은 통계분석이 아니라 **3D 구조 시뮬레이션** — 단백질 구조(PDB)·리간드 3D·힘장·검색상자 준비가 필요.
- CB-Dock2는 이 과정을 웹에서 자동화 → 논문은 클릭만 함(코드 없음).
- 로컬 재현은 Vina·OpenBabel 설치 + 구조 준비가 필요해 앞 단계(R 스크립트)보다 셋업이 무겁다.
- 그래서 우리는 **① 코드(Vina) + ② 웹(CB-Dock2)** 를 모두 제공해 상황에 맞게 선택하게 함.

---

## 3. 기대 결과 / 판정
| 표적 | 논문 (CB-Dock2) | Vina 재현 기대 |
| --- | --- | --- |
| FN1 (3m7p) | −7.0 kcal/mol | < −5.0 (강한 결합) 재현되면 성공 |
| ALDH2 (8dr9) | −9.8 kcal/mol | < −5.0 (강한 결합) 재현되면 성공 |

→ 절대값이 논문과 소수점까지 같을 필요는 없고, **두 표적 모두 강한 결합(−5 미만)** 이 나오면 "레스베라트롤 이중 표적" 결론 재현.

## 4. 한계 / 주의
- 로컬 Vina 점수 ≠ CB-Dock2 점수 (알고리즘 차이) — 절대 비교 금물, 경향으로 판단.
- blind docking 박스가 크면(대형 단백질) 정밀도 하락 → `exhaustiveness` 상향 또는 활성부위 박스로 조정 가능.
- 샌드박스(네트워크·도구 없음)에선 실행 불가 → **사용자 PC에서** README대로 실행.
