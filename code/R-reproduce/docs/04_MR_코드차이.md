# 04. MR — 저자 원본 코드 vs 우리 재현 코드

업로드된 6개 파일을 **1↔1로 직접 대조**한 결과.

| 저자 원본 (뒤 숫자) | 우리 재현 (앞 숫자) | 역할 |
| --- | --- | --- |
| `Mendelian randomization 1.R` | `01_mr_run.R` (+`R/mr_func.R`) | 노출→결과→harmonise→mr→민감도 |
| `Mendelian randomization 2.R` | `02_mr_filter.R` | IVW 필터 (인과 유전자 선별) |
| `Mendelian randomization 3.R` | `03_mr_forest.R` | forest plot |

**결론 먼저:** 핵심 MR 통계 로직(harmonise → IVW·Egger·weighted median → OR → 민감도 4종 → IVW 필터 → forest)은 **저자와 완전히 동일**. 차이는 전부 **① 노출 도구변수 준비 방식 ② 실행 편의(경로·함수화·반복)** 두 갈래이고, 우리는 여기에 **③ 검증 단계(DEG 확인·Supp9 대조·Figure 3D 조합)**를 추가했다.
✅ 결과 = **논문 Supplementary Table 9 IVW와 완전 일치** (후보 유전자 10개 전부, FN1 OR 2.78 · ALDH2 OR 0.67 포함).

## 1. MR1 ↔ 01_mr_run.R (+R/mr_func.R)

### ① 노출(exposure) 준비 — ★ 유일한 "본질적" 차이

**저자 MR1.R**
```r
exposureFile = "exposure.F.csv"                 # 이미 만들어둔 도구변수(IV) 파일
exposure_dat = read_exposure_data(filename=exposureFile, sep=",",
                 snp_col="SNP", beta_col="beta.exposure", ...)   # clump=FALSE
```
- 저자는 **미리 만들어 둔 IV CSV**를 그대로 읽음. 이 파일을 어떻게 만들었는지(어떤 유전자, 어떤 clump)는 코드에 없음.

**우리 01_mr_run.R**
```r
s8 <- data.table::fread(SUPP8, skip = 1, header = TRUE)   # 저자 Supplementary Table 8
exposure_dat <- TwoSampleMR::format_data(as.data.frame(s8), type = "exposure",
    snp_col="SNP", beta_col="beta.exposure", se_col="se.exposure",
    effect_allele_col="effect_allele.exposure", ...,
    phenotype_col="exposure")                              # 후보 유전자 10개
write.csv(exposure_dat, ".../exposure.candidates10.csv")
```
- **왜 다른가?** 오프라인이라 LD clumping API를 못 쓴다 → 저자가 논문에 **이미 clump해 공개한 도구변수(Supplementary Table 8)**를 노출로 사용. 이렇게 하면 저자와 **정확히 같은 SNP 집합·같은 통계**로 출발 → 논문 재현 정확도 100%.
- 저자는 IV를 `exposure.F.csv`로 숨겼지만, 우리는 그 출처(Supp8)를 명시하고 유전자 **10개 전부**를 한 번에 노출로 넣음.
- 참고: `R/mr_func.R`에는 eQTL VCF에서 beta/se/eaf를 직접 뽑는 `parse_vcf()`도 있음(교차검증용). 다만 **01의 실제 실행 경로는 Supp8 직접 사용**이다.

### ② harmonise + MR 실행 — ★ 완전 동일 (`R/mr_func.R::run_mr`)

**저자 MR1.R**
```r
dat = harmonise_data(exposure_dat, outcomeData)
dat = dat[dat$pval.outcome > 5e-06, ]                 # 역인과/다면발현 방지
mrResult <- mr(dat, method_list=c("mr_ivw","mr_egger_regression","mr_weighted_median"))
mrTab = generate_odds_ratios(mrResult)
mr_heterogeneity(dat); mr_pleiotropy_test(dat)
mr_singlesnp(dat);     mr_leaveoneout(dat)
```
**우리 run_mr()**
```r
dat <- harmonise_data(exposure_dat, outcome_dat)
dat <- dat[!is.na(dat$pval.outcome) & dat$pval.outcome > 5e-06, ]   # 동일 (+NA 방어)
res <- mr(dat, method_list=c("mr_ivw","mr_egger_regression","mr_weighted_median"))
mrTab <- generate_odds_ratios(res)
mr_heterogeneity / mr_pleiotropy_test / mr_singlesnp / mr_leaveoneout  # tryCatch로 감쌈
```
- **방법·임계값 전부 동일.** 우리 쪽은 `NA` 방어와 `tryCatch`만 추가.

## 2. MR2 ↔ 02_mr_filter.R — ★ 로직 완전 동일

**저자 MR2.R**
```r
for(geneName in unique(rt$exposure)){
  geneData = rt[rt$exposure==geneName,]
  if(nrow(geneData)==3){                                        # 3방법 다 있고
    if(geneData[method=="Inverse variance weighted","pval"]<0.05){   # IVW p<0.05
      if(sum(or>1)==nrow | sum(or<1)==nrow){                   # 3방법 OR 방향 일치
        ivw = rbind(ivw, geneData) }}}}
pleRT = pleRT[pleRT$pval>0.05,]                                 # 다면발현 없는 것만
outTab = ivw[ivw$exposure %in% pleRT$exposure,]
```
**우리 02_mr_filter.R** — `filter_one(tag)` 안에 **동일 3중 조건** 그대로:
```r
if (nrow(gd) == 3) {
  if (gd[gd$method=="Inverse variance weighted","pval"] < 0.05) {
    if (sum(gd$or>1)==nrow(gd) | sum(gd$or<1)==nrow(gd)) ivw <- rbind(ivw, gd) }}
ple <- ple[ple$pval > 0.05, ]; ivw <- ivw[ivw$exposure %in% ple$exposure, ]
```
- **차이는 껍데기뿐**: 저자는 setwd 후 파일 하나, 우리는 `filter_one()`을 `FinnGen`·`GCST` 두 tag로 반복 + 파일 없을 때 방어. 판정 로직은 글자 단위로 동일.

## 3. MR3 ↔ 03_mr_forest.R — 로직 동일, 스타일 간소화

| 항목 | 저자 MR3.R | 우리 03_mr_forest.R |
| --- | --- | --- |
| 입력 | `dir()`로 폴더 내 **모든 csv** 읽어 rbind | tag별 `table.MRresult.<tag>.csv` 지정 |
| 방법 | `selectMethod="Inverse variance weighted"`만 | 동일 (IVW 행만) |
| 표 컬럼 | 공백 spacer + `OR(95% CI)` + pval 포맷 + exposure 중복제거 | 동일 (spacer 폭 12) |
| 테마 | `base_size=18`, **유전자별 박스 색상**(`boxcolor` 팔레트, 행마다 `edit_plot`), **유의 pval 볼드**, 테두리 | `base_size=14`, 단색·볼드 없음 (간소화) |
| xlim | 고정 `c(0,2)` | 동적 `max(3, ceil(max uci))` (FigureD는 `c(0,2)`) |
| 출력 | `forest.pdf` | `forest_MR_<tag>.png` |
| 패키지 | 설치 가정 | 없으면 `install.packages` 자동 |

**개념은 동일**(IVW 결과를 forestploter로). 저자 그림이 색·볼드로 더 화려하고, 우리는 재현 확인용으로 단색 PNG.

### ★ 우리만 추가: 논문 Figure 3D 조합 (`forest_figureD`)
저자 MR3은 "한 폴더에 모아둔 csv들"을 rbind해 그리는 방식 — 즉 **필터 통과 유전자들을 폴더에 모으면 그게 곧 Figure 3D**. 우리는 이 과정을 코드로 명시:
```r
# FinnGen 통과 7개 + GCST 통과 3개 = 논문 D의 10개
for (tag in c("FinnGen","GCST")) d <- read.csv(paste0("IVW.filter.",tag,".csv"))  # 통과분만
D <- do.call(rbind, parts); D <- D[order(D$exposure), ]     # 알파벳순(논문 D와 동일)
forest(...) → forest_MR_FigureD.png
```
→ 논문 Figure 3D의 **인과 유전자 10개 forest를 그대로 재현**.

## 4. 결과 — 논문과 완전 일치

우리 **GCST outcome IVW** = 논문 Supp9와 **b·se·p·nsnp까지 동일** (후보 10개 전부 대조):

| 유전자 | nsnp | OR (95% CI) | p | 방향 | 논문 Supp9 |
| --- | --- | --- | --- | --- | --- |
| **FN1** | 3 | **2.777** (1.25–6.17) | 0.0122 | 위험↑ | b=1.021, p=0.0122 **일치** |
| **ALDH2** | 14 | **0.673** (0.47–0.97) | 0.0321 | 보호↓ | b=−0.395, p=0.0321 **일치** |

→ 저자 도구변수(Supp8)를 그대로 써서 **논문 재현 정확도 100%**. 나머지 8개 유전자도 `compare_paper_vs_ours.MR_IVW.csv`에서 일치 확인.

## 5. 한 줄 요약
> 저자 3파일(MR1 실행·MR2 필터·MR3 forest)을 **동일하게 3파일(01/02/03)로** 재현. harmonise·IVW·Egger·median·민감도·IVW필터·forest **로직은 글자 단위로 동일**. 실질 차이는 **노출 IV 준비**(저자=완제 `exposure.F.csv` / 우리=논문 Supp8 IV를 `format_data`)뿐이며 이는 오프라인 LD clumping 우회용. 나머지 차이는 경로 상대화·함수화·2개 outcome 반복 같은 편의 개선이고, 여기에 **DEG 확인·Supp9 자동대조·Figure 3D 조합**을 더했다. 결과는 **논문 Supp9와 완전 일치**.
