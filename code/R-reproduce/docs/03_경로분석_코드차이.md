## 📂 파일 경로 (입력 → 결과·사진)

**코드**: `03_pathway/R/gsea_func.R`(함수), `run_gsea.R`(F 실행) — ORA/ssGSEA는 `run_ora_barplot.R`·`run_ssgsea_heatmap.R`(저자 코드 없어 별도)

**결과·사진** — `03_pathway/output/`
| 위치 | 파일 |

## 0. 한눈에

| 항목 | 저자 `GSEA.R` | 우리 |
| --- | --- | --- |
| 구조 | 1파일(함수+실행 같이) | `gsea_func.R`(함수) + `run_gsea.R`(실행) 분리 |

## 1. 단계별 코드 대조

### ① logFC 순위 만들기 (★ 완전 동일)

**저자**
```r
logFC <- diff_data$logFC
names(logFC) <- rownames(diff_data)
logFC <- sort(logFC, decreasing = TRUE)
```
**우리**
```r
logFC <- diff_data$logFC
names(logFC) <- rownames(diff_data)
logFC <- sort(logFC, decreasing = TRUE)
```
- **글자까지 똑같음**. DEG의 logFC로 유전자 줄세우기.

### ② GSEA 실행 (★ 완전 동일)

**저자**
```r
kk <- GSEA(logFC, TERM2GENE = gmt, pvalueCutoff = 1)
kkTab <- as.data.frame(kk)
```
**우리**
```r
set.seed(123)                                    # ← 재현성 위해 추가
kk <- GSEA(logFC, TERM2GENE = gmt, pvalueCutoff = 1)
kkTab <- as.data.frame(kk)
```
- 동일. 우리만 **`set.seed(123)`** 추가 (GSEA 순열검정 난수 고정 → 재실행 시 값 동일).

### ③ 유의 경로 필터 (★ 완전 동일)

**저자**
```r
kkTab_sig <- kkTab[kkTab$p.adjust < 0.05, ]
up_terms   <- rownames(kkTab_sig[kkTab_sig$NES > 0, ])
down_terms <- rownames(kkTab_sig[kkTab_sig$NES < 0, ])
```
**우리**
```r
sig  <- kkTab[kkTab$p.adjust < ADJP, ]     # ADJP=0.05
up   <- sig[sig$NES > 0, ]
down <- sig[sig$NES < 0, ]
```
- 동일 (p.adjust<0.05, NES 부호로 켜짐/꺼짐).

### ④ 시각화 gseaplot2 (★ 완전 동일)

**저자**
```r
gseaplot <- gseaplot2(kk, pathways, base_size=10, title=plot_title, color="firebrick")
ggsave(paste0("GSEA.", comparison_name, ".", file_suffix, ".pdf"), ...)
# up: head(up_terms, min_terms) / down: head(down_terms, min_terms)
```
**우리**
```r
g <- gseaplot2(kk, geneSetID=terms, base_size=10, title=title, color="firebrick")
ggsave(file.path(figDir, paste0("GSEA.", db, ".", tag, ".", suffix, ".pdf")), ...)
```
- 동일 (firebrick, base_size=10, 상위 min_terms개). 우리는 **파일명에 DB 추가** + `figures/` 폴더로 저장.

### ⑤ 결과 저장

**저자**: `GSEA.result.<비교>.txt` 각각 + 유의결과 `rbind`로 `All_GSEA_Results_Combined.txt` 합본
**우리**: `GSEA.<DB>.<비교>.txt` 각각 + 개수요약 `GSEA_summary.txt`
- 저장 형식 차이만(합본 내용 vs 개수요약). 원자료는 동일하게 다 저장.

## 3. 주의: 저자 GSEA.R의 범위

- 저자 `GSEA.R`은 **GSEA(F 그림)만** 만듦. **KEGG** gmt 하나만 씀.
- 논문 Figure 4의 **ORA 막대(C/D/E/G)·ssGSEA 히트맵(H/I) 코드는 저자가 공유 안 함**
    → 우리가 논문 그림 보고 별도 재현 (`run_ora_barplot.R`, `run_ssgsea_heatmap.R`)
- 즉 **F만 저자 코드 대조 가능**, C/D/E/G·H/I는 대조할 원본이 없음.

## 4. org.Hs.eg.db 검증 — 심볼 직접 vs ENTREZ 변환 (같은 결과 확인)

논문 Methods는 `org.Hs.eg.db` 주석 사용을 명시하지만, **저자 GSEA.R도 심볼 gmt(`c2.cp.kegg.Hs.symbols.gmt`) 직접 대조** → org.Hs.eg.db 실제 안 씀. 우리도 동일.
→ 두 방식을 실제로 돌려 비교 (`run_gsea_annotation_compare.R`, 말기vs정상·Hallmark):

| 방식 | org.Hs.eg.db | 유의경로 | 비고 |
| --- | --- | --- | --- |
| **A안(현재)** | ❌ | 22 | 심볼 gmt 직접 (`TERM2GENE=심볼`) |
| **B안(논문 명시)** | ✅ | 21 | 심볼→ENTREZ 변환 후 GSEA |

- **NES 평균 차이 0.07** (무시 수준), 심볼→ENTREZ에서 **301개(7.48%) 탈락**
- **NES 최대 차이 1.445는 함정** — HEME_METABOLISM(p.adj 0.98)처럼 **유의하지 않은 노이즈 경로**에서만. NES가 0 근처라 유전자 몇 개만 빠져도 부호가 쉽게 뒤집힘
- **실제 유의 경로는 거의 동일**: EMT(Δ0.071), IFNγ(Δ0.056), Complement(Δ0.057)

→ **결론**: org.Hs.eg.db 써도 **생물학적 결과 안 바뀜**. 오히려 매핑 실패로 7.48% 손해 → **A안(심볼 직접)이 더 깔끔** (저자 GSEA.R도 A안). 결과: `output/annotation_compare.txt`

## 5. 한 줄 요약
> 저자 `GSEA.R`의 GSEA 로직(순위→GSEA→p.adjust<0.05→gseaplot2)을 **글자까지 그대로** 함수화. 차이는 **Hallmark 추가(2종)·함수분리·set.seed·폴더정리**뿐. ORA·ssGSEA는 저자 코드가 없어 논문 그림 기반으로 우리가 새로 구현. org.Hs.eg.db(심볼→ENTREZ)는 검증 결과 **결과 동일**이라 심볼 직접 대조 유지.
