# 3. R 코드 흐름 분석 (왜 이렇게 처리했나)

전체 흐름:
```
데이터 입력 → 전처리 → 정규화 → DEG 분석 → 검증(ML/ROC) → MR·eQTL·GWAS 연계 → 결과 저장
```
아래는 각 R 스크립트가 이 흐름의 어느 칸을 맡는지 + **왜** 그 처리를 하는지 쉬운 설명.

---

## ① 데이터 입력 + 전처리 — `data preprocessing 1.R`
**하는 일**: GEO series matrix(probe 단위 발현) 읽기 → GPL 주석으로 **probe ID를 유전자 심볼로 변환** → 같은 유전자에 매핑된 여러 probe는 `avereps`로 평균.

**왜?**
- 마이크로어레이/일부 플랫폼은 값이 "유전자"가 아니라 "probe(탐침)" 단위 → 논문/생물학적 해석은 유전자 단위여야 하므로 변환 필수.
- 한 유전자에 probe가 여러 개면 값이 중복 → **평균**으로 하나로 합쳐야 이후 분석에서 유전자당 1행이 됨.

---

## ② 정규화 + 그룹 라벨링 — `data preprocessing 2.R`
**하는 일**: 발현값 분포 보고 **필요 시 자동 log2 변환** → `normalizeBetweenArrays`(quantile 정규화) → Control/Early/Late 3군으로 샘플 분리하고 열 이름을 `{샘플}_{그룹}`으로 라벨.

**왜?**
- **log2**: RNA 발현은 값의 범위가 매우 넓음(수~수만) → 로그로 눌러줘야 통계가 정규분포 가정에 가까워지고 fold-change 해석이 쉬움. 값이 이미 로그면 건너뜀(자동 판정).
- **quantile 정규화**: 샘플마다 전체 신호 세기(스케일)가 다름 → **모든 샘플의 발현 분포를 동일하게 맞춰야** 샘플 간 비교가 공정해짐. 안 하면 "생물학적 차이"가 아니라 "측정 스케일 차이"를 잡아냄.
- **그룹 라벨**: 이후 DEG·ML이 열 이름에서 그룹을 읽음 → 라벨링이 파이프라인의 약속.

---

## ③ 배치보정 병합 — `data preprocessing 3.R`
**하는 일**: 여러 데이터셋의 **교집합 유전자**만 남겨 합치고 → `sva::ComBat`으로 **배치효과(batch effect) 제거**.

**왜?**
- 서로 다른 GEO 데이터셋은 실험 장비·시기·기관이 달라 **데이터셋 고유의 체계적 차이(배치효과)**가 있음.
- 그냥 합치면 "어느 데이터셋 출신인지"가 최대 변수로 잡혀 진짜 질병 신호를 가림 → ComBat이 데이터셋별 평균/분산 차이를 통계적으로 보정해 **한 실험처럼** 만듦.
- 교집합 유전자만 쓰는 이유: 데이터셋마다 측정 유전자 집합이 달라, 공통 유전자라야 나란히 비교 가능.

---

## ④ DEG(차등발현) 분석 — `differential expression analysis.R`
**하는 일**: `limma`의 `lmFit → eBayes → topTable`로 Control vs DKD **유전자별 차이 검정** → `|log2FC|>0.585 & adj.P<0.05`로 유의 유전자 필터 → 화산도/히트맵.

**왜?**
- 수천~수만 유전자를 **동시에** 두 군 비교 → 유전자마다 t-test 하되, `limma`의 **eBayes(경험적 베이즈)**는 유전자별 분산을 전체에서 빌려와 안정화 → 샘플 수가 적어도 신뢰도 높은 검정.
- `log2FC>0.585` = **1.5배 이상** 변화, `adj.P<0.05` = **다중검정(BH) 보정** 후 유의 → 우연한 다수 검정에서 거짓양성을 통제.
- 화산도/히트맵: 어떤 유전자가 얼마나·어느 방향으로 변했는지 **눈으로 확인**(논문 그림).

## ④' 경로 해석 — `GSEA.R`
**하는 일**: 유전자를 logFC로 정렬해 `clusterProfiler::GSEA`(KEGG)로 **어떤 생물학적 경로(pathway)가 통째로 위/아래로 움직였는지** 검정.

**왜?**: 개별 유전자가 아니라 **경로 단위**로 봐야 "무슨 생물학적 기능이 DKD에서 바뀌었나"를 해석 가능(섬유화·대사 등).

---

## ⑤ 검증 = 바이오마커 선별 + 성능 — `machine learning modeling 1·2.R`
**하는 일(1)**: 후보 유전자에 대해
- **LASSO**(`glmnet`, 10-fold CV): 예측에 기여 안 하는 유전자 계수를 0으로 → 소수 핵심만 남김.
- **SVM-RFE**(`caret::rfe`): 성능 기여 낮은 유전자를 반복 제거하며 최적 소수 특징 선택.
- **Venn 교집합**: 두 방법이 **공통으로** 뽑은 유전자만 최종 후보(→ FN1/ALDH2류).

**하는 일(2)**: 최종 유전자별로 **ROC/AUC + bootstrap 95% CI**를 **훈련셋·검증셋 모두**에서 계산.

**왜?**
- DEG만으로는 "차이 있다"까지 → **진단 바이오마커**로 쓰려면 "이 유전자로 환자/정상을 구분할 수 있나(AUC)"를 증명해야 함.
- 두 서로 다른 방법의 **교집합**을 쓰는 이유: 한 방법의 과적합/편향을 걸러 **robust한 소수 마커**만 남기려고.
- **독립 검증셋 ROC**: 훈련에서 좋아 보인 게 새 데이터에서도 되는지 → 일반화 확인(과적합 방지).

---

## ⑥ MR·eQTL·GWAS 연계 — `Mendelian randomization 1·2·3.R`
**하는 일**:
- (1) **노출=유전자 eQTL**(발현 조절 SNP), **결과=DKD GWAS**(FinnGen) → `TwoSampleMR`로 두 요약통계를 harmonise 후 MR(IVW 등) 실행.
- (2) IVW p<0.05 + 세 방법 OR 방향 일치 + **다면발현(pleiotropy) p>0.05** 인 유전자만 통과.
- (3) 통과 유전자 IVW 결과를 **forest plot**으로.

**왜?**
- 상관/차등발현은 "관련 있다"까지 → **인과(원인)**인지는 모름. MR은 유전 변이(무작위 배정에 가까움)를 도구변수로 써서 **"유전적으로 이 유전자 발현이 높으면 DKD 위험이 오르나"** 라는 **인과 방향**을 추정.
- **여러 MR 방법 방향 일치 + pleiotropy 배제**: MR의 핵심 가정(도구변수가 다른 경로로 결과에 영향 X)을 어긴 유전자를 걸러 **거짓 인과**를 배제 → 치료 타깃으로서 신뢰도↑.

## ⑥' 조직 수준 근거 — `single-cell RNA-seq analysis 1~4.R`
- **1** Seurat + harmony 통합·클러스터링·세포유형 주석: 마커가 **어느 신장 세포**에서 나오는지.
- **2** cell ratio 막대그래프: 군(Late 등)별 **세포 구성 변화**.
- **3** monocle pseudotime: 근위세뇨관(PCT→dPCT) **손상 궤적**을 따라 발현 변화.
- **4** CellChat: 세포 간 **신호전달(통신)** 변화.

**왜?**: bulk는 조직 전체 평균 → **어느 세포가 원인인지** 모름. scRNA로 "FN1/ALDH2가 특정 세포에서, 손상 경로를 따라, 특정 신호로 작동"을 보여 **기전적 근거**를 채움.

다음 단계 입력이자 논문 그림·표 소스가 됨.

### 한 줄 요약
> **차이(DEG) → 핵심 소수 선별(ML) → 진단력 증명(ROC) → 인과 확인(MR) → 세포기전(scRNA)** 로,
> "관련성 → 예측력 → 인과성 → 기전"을 단계적으로 쌓아 FN1·ALDH2를 **진단 바이오마커이자 치료 타깃**으로 결론내는 구조.
